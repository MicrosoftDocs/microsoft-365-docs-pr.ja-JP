# <a name="get-started-driving-adoption-of-microsoft-viva-topics"></a>Microsoft の Microsoft のトピックの導入の推進を開始する

導入を開始する前に、知識管理と「多くのトピック」に含まれる概念を理解する必要があります。 次の図は、トピックの検出と管理の間に何が起こるかを示しています。

![多くのトピックのアーキテクチャ](../media/knowledge-management/topic-management-architecture.png)

- **検出**: ユーザーは、トピック カードを使用して、毎日使用するアプリの知識を検出できます。また、Microsoft Search のトピックを見つけすることもできます。
- **Curation**: 対象分野の専門家 (SMEs) はトピック ページを通じてトピックを絞り込み、AI はトピックの入力から学習します。 トピック センターには、ユーザーが参照できるトピック ページと、専門家が管理できるトピック ページが含まれています。
- **識別**: Microsoft Graph と人工知能 (AI) の知識とユーザー (トピック、スキルなど) を特定し、関連するトピックに自動的に整理します。 SharePoint コンテンツは、セキュリティ コンテンツでインデックス付けされます。
- **拡張機能**: Microsoft Graph コンテンツ コネクタ (近日公開予定) では、外部サービスとデータ リポジトリから知識を取り込めます。

詳細については、概要 [を確認してください](topic-experiences-overview.md) 。

次の注意が必要です。

- トピックの検出は、より多くのコンテンツが利用可能になったときに改善されます。
- 新しいエクスペリエンスで情報が表示される場合でも、データのセキュリティ、プライバシー、および場所は保持されます。
- ユーザーは、多くのトピックを表示するためにライセンスが必要です。
- 検出は、最初は英語のコンテンツです。

準備を行う場合は、次の点を確認してください。

- トピックの検出にどのコンテンツを使用する必要がありますか。
- トピックを管理するユーザー
- トピック カードとハイライトは誰に表示されますか?
- どのトピックが必要ですか?

以下の一覧で、多くのトピックを利用するための前提条件を確認してください。

|製品または機能 |説明 |
|:-------|:--------|
|SharePoint Online とモダン SharePoint ページ |トピック マイニングには SharePoint サイト上のコンテンツだけが含まれます。トピック カードはモダン ページにのみ表示できます。|
|Microsoft Graph |Microsoft Graph の設定を使用して、検索または Delve からトピックを含めるか除外するか制御できます。 |

## <a name="plan-for-adoption"></a>導入を計画する

多くのトピックの導入を計画するには、次の必要があります。

![導入を計画する手順](../media/knowledge-management/km-adoption-plan-adoption.png)

1. アプローチとターゲット シナリオを計画します。
    - シナリオの定義と優先順位付けについて考 [えます](#target-scenarios)。
    - 関係者と [、関与する](#identify-stakeholders) 必要があるプロジェクト チーム のメンバーについて考える。  
    - ビジネスに与える影響と、成功を測定する方法 [を把握します](#create-a-success-plan)。

2. 組織を引き付け:
    - 関与する必要があるビジネス グループとビジネス チームを特定し、それらのビジネス グループと計画しているシナリオ全体を調整します。
    - 初期の導入者に取り組み、最も重要な早期フィードバックを得て、最善のソリューションを得る方法について考え始める必要があります。
    - コミュニティの構築を開始し、これらの異なるグループが組織全体でどのように多くのトピックを使用できるのかについて考えてみください。

3. 組織のトレーニング: ほとんどのユーザーは、トピックの概念とトピック カードが関連情報をコンパイルする方法を直感的に理解し、その価値を理解して確認します。 しかし、自分の文化や組織に合わせて調整されたトレーニングを作成し、多くのトピックを使用する方法を示したい場合があります。 いくつかのトレーニング リソース:
    - [Project のリソース センターです](https://aka.ms/projectcortex)。 概要と機能情報、記録されたオフィス時間のビデオとプレゼンテーション、パートナーとその提供物に関する情報が含まれます。
    - 近日公開予定、エンド ユーザー向けトレーニング ビデオとヘルプ。

4. [チャンピオン ネットワークを構築します](#build-a-champion-network)。
    - 実践ネットワークやチャンピオン ネットワークのコミュニティが既に存在する可能性があります。 これらは、互いに助け合い、同僚を互いに助け合い、それを助け合う良い方法です。 また、成功事例を共有できます。これは非常に有用です。 アドバイスを提供し、楽しい思いを生み出します。

### <a name="target-scenarios"></a>ターゲット シナリオ

トピックを使用して成功するには、組織内でどのように多くのトピックを使用する必要があります。 知識管理とトピックが組織に役立ついくつかのシナリオを以下に示します。

- 役割オンボーディング &トレーニング: 新しい組織の用語、主要なプロジェクト、および文化を理解することが、オンボーディングの重要なステップです。 トピックを簡単に見つけることができるのは、新しい従業員が新しい仕事、役割、またはプロジェクトにすばやく取り組むのに役立ちます。
- 専門知識の検索と情報共有: トピックを管理および共有すると、組織のユーザーは、毎日の作業で役立つ情報や専門家を簡単に見つけることができる。
- 意思決定の拡大と市場への時間の改善: 情報や専門家が簡単にアクセスできると、プロジェクトをより簡単に決定し、時間を節約できます。

たとえば、役割オンボーディングのシナリオを次に示します。

- 従業員 (ヨルダン) が新しい役割を引き受け、または新しく採用され、役割から始まるばかりです。 ヨルダンは、できる限り早く関与して生産性を高めしたいと考えています。 しかし、ヨルダンには開始場所を見つける際の支援も必要です。
- ヨルダンが新入社員や情報を探している他のユーザーを支援するトピック ページを作成する前に役割を果たした同僚 (Kim)。
- Kim は SME であり、未確認のトピック ページを表示するアクセス許可を持っていた。 未確認のトピック ページは、AI が発見して作成した内容の開始点として最適であり、Kim は編集して専門家のリソース、定義、その他のリソースを固定することができます。
- ヨルダンが SharePoint の新しい投稿を読み上げると、トピックのハイライトが表示され、その上にカーソルを合わせると、用語の定義と、他の質問に問い合わせるユーザーをすばやく取得できます。 以前、ヨルダンは、この情報を探し回り、同僚に連絡して何か質問する必要があるユーザーを確認する必要があった可能性があります。
- トピックを通じてこの情報を公開すると強力な場合があります。これは、この情報は以前に提供されている可能性があるが、サイロ化され、見つけるのが難しい可能性があるからである。 ヨルダンが使用しているアプリケーションに組み込み、これらの専門家を見つけるのを支援すると、エンゲージメントとコミュニティの感覚を高め合う可能性があります。 また、新しい役割を扱う際に、より権限が強化されたと感じるのにも役立ちます。

シナリオを特定した後、次のシナリオに優先順位を付けできます。

優先順位を付ける方法の 1 つは、影響と実装の容易さを示すグリッドでシナリオをプロットアウトする方法です。 影響が大きく、実装しやすいシナリオを探し、それらのシナリオを最優先にします。 影響が低く、実装が難しいシナリオが最も低い優先度です。 大きな影響を与え、実装しやすいシナリオで迅速に勝つ場合は、ユーザーが楽しみを感じ、トピックを使用する可能性を確認するのに役立ちます。

![影響が大きく、シナリオの実装が容易で優先度が高い](../media/knowledge-management/topics-prioritize-scenarios.png)

最初に重点を置くいくつかの主要なシナリオを選択し、早期導入者と一緒にフィードバックを得てから、ステージにロールアウトします。 これにより、反復処理を行い、改善を行い、フィードバックを得て、時間のたつにつれて導入を拡大することができます。 

### <a name="identify-stakeholders"></a>関係者の特定

プロジェクトの関係者を特定します。 主な役割は、エグゼクティブ スポンサー、成功所有者、およびチャンピオンです。

|ロール |責任 |部門 |
|:-------|:-------|:--------|
| エグゼクティブ スポンサー   | 会社に高レベルのビジョンと価値を伝える   |  エグゼクティブ リーダー   |
| プロジェクト リード | 起動の実行とロールアウトプロセス全体を監視する | プロジェクト管理 |
| ナレッジ管理者| Set up and configure Viva Topics | IT 部門 |
| ナレッジ マネージャー | トピックを管理し、分類を監視する | 任意の部署 |
| 分類マネージャー | 分類を監視する | 任意の部署 |
| 対象分野の専門家とトピックの投稿者 | トピックと説明を生成または確認する | 任意の部署 |
| チャンピオン | オブジェクト処理の助けと管理 | 任意の部署 (スタッフ) |
| テナント管理者 | テナント レベルの設定を構成する | IT 部門 |
| Power Platform 管理者| 共通データ サービス環境を構成する | IT 部門 |
| 検索管理者またはマネージャー | 検索の設定を構成する | IT 部門 |

大規模な組織では、これらの役割に複数のユーザーが含まれた場合も、それらの間の調整を行う必要があります。 小規模な企業では、1 人のユーザーがこれらの役割のいくつかを実行する場合があります。 プロジェクトの異なるフェーズでは、さまざまなロールが関与する可能性があります。 たとえば、テナント管理者は機能の設定に関わる人が多く、トピックの定義を始めるまで、対象分野の専門家とチャンピオンは関与しなきます。
 
ロールアウトを通じてこれらの各役割を満たすことをお勧めしますが、特定のソリューションを開始するためにそれらのすべての役割を必要としない場合があります。

### <a name="create-a-success-plan"></a>成功計画を作成する

これらのインジケーターを使用して、組織内の多くのトピックの成功を測定します。 見てください：

1. トピックの使用法:
      - トピックのインプレッション数
      - トピックの数量 – 確認済みと未確認の両方が、選択したトピックの一覧に表示されます。
      - 発行されたトピック ページの数。
1. トピック カードからのエンドユーザー フィードバック。
1. 従業員の満足度調査を行います。 「トピック」では、従業員が情報を見つける能力を向上させる必要があります。そのため、そのエクスペリエンスに関する情報やフィードバックを収集する方法を見つける必要があります。
1. 検索分析に良い影響を与えます。 トピックは検索エクスペリエンスに表示されるので、時間がたつと、検索でトピックを見つけやすいので、破棄された検索の率が低下する可能性があります。 

### <a name="build-a-champion-network"></a>チャンピオン ネットワークを構築する

組織内にチャンピオン ネットワークを構築します。 チャンピオンは次の可能性を持つため重要です。

- チーム内に影響を与える円を作成する
- トピック管理とメンテナンス&する

異なるロール (ナレッジ マネージャーと対象分野の専門家) からチャンピオンを募集できます。

多くのチャンピオン ネットワークでは、プラットフォームYammerを使用しています。 このYammer、ユーザーは質問を投稿して回答を得たり、成功事例を共有することができます。 この言葉を単独で説明するのは難しいので、同僚にアドバイスを提供し、他のチームが自分のシナリオについて考えできるよう、同僚にアドバイスを提供し、

一部の組織では、ハッカー (公式または非公式、仮想的、または対人) を使用して、特定のプロジェクトに取り組む人々のグループを収集しています。 たとえば、該当分野の専門家を集めて、一連のトピック ページの作成に一緒に取り組む場合があります。

チャンピオンを認識する方法について考えてみる。 アクティビティに報酬を与え、ある程度の認識を与え、見えるコミュニティの感覚とエンゲージメントを生み出して、何かに貢献しているような気がして、投資から何かを取り戻すようになります。

ロールアウトの準備が整ったので、継続的な関与を促す必要があります。

- チャンピオンのためにYammerグループを維持します。
- 成功事例を共有します。
- 定期的にエンゲージメント イベントをホストして、ストーリーを共有したり、新機能を紹介したりします。
- ユーザーの課題を設定し、競争を実行します。

## <a name="next-steps"></a>次の手順

多くのトピックを展開する準備ができたら、ユーザーを巻き込む必要があります。

- 機能セットの導入を開始し、シナリオについて考えてみます。
- 関係者を収集し、シナリオを作成します。
- コミュニティを駆動し、どのように参加を行うのか考えます。
- その後、準備手順を完了します。 技術的な準備が必要な場合や、業務の準備が必要な場合があります。
- 最後に、ソーシャル化と昇格を行います。
