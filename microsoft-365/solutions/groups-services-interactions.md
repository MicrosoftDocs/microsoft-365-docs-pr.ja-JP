# <a name="groups-services-interactions"></a>グループサービスの相互作用

Microsoft 365 グループは、Microsoft 365 プラットフォーム内のさまざまなサービスとワークロードに共通の fabric を提供して、エンドユーザー向けの接続性を提供しています。 中核となるのは、Microsoft 365 グループが存在するためです。次の情報が提供されます。

- メンバーシップを管理する方法 (Azure AD)
- メッセージングと会話を行う場所 (Exchange メールボックス、Microsoft Teams、Yammer)
- ファイルを保存する場所 (SharePoint)
- スケジュール設定用の予定表 (Exchange)
- ノートをキャプチャするためのノートブック (OneNote)

グループの作成時点では、次のようないくつかのリソースもプロビジョニングされていますが、サービスから初めてアクセスしたときには表示されません。

- グループタスクを管理するためのボード (プランナー)
- レポート用のワークスペース (Power BI)
- 共有ビデオの領域 (Microsoft Stream)
- 共有フォームの領域 (フォーム)

Microsoft 365 では、他のサービスが Microsoft 365 グループと対話して、グループメンバーに追加の機能を提供することができます。
この例には、次のようなものがあります。

- アプリ用のパワーアプリ
- ワークフローの電源自動化
- Web 上のプロジェクトとウォーターフォールベースのプロジェクト管理のロードマップ
- チャネルベースの会話の Teams
- 関心のあるコミュニティの Yammer

## <a name="user-interactions-with-groups"></a>グループを使用したユーザー操作

Microsoft 365 グループは、管理者とエンドユーザーの両方から、さまざまなインターフェイスから作成および管理できます。 

### <a name="administrative-experiences"></a>管理者エクスペリエンス

管理者は、いくつかのワークロード管理センター、スクリプトをサポートするコマンドラインインターフェイス、および Graph API と対話するカスタムのアプリを、Microsoft 365 グループの作成と管理を行うことができます。 唯一の例外は Yammer グループで、Yammer の web インターフェイスから作成する必要があります。

**関連する設定**

グループ設定を管理できるさまざまな管理インターフェイスでは、いくつかの重なりがあり、注意する必要があります。

**Microsoft 365 管理センター**

Microsoft 365 管理センターでは、グループへのゲストアクセスは既定で有効になっており、ゲストの追加を所有者に許可することができます。 この管理センターから、グループに対して使用可能なその他の組織レベルの制御はありません。

**Azure AD 管理センター**

Azure AD 管理センターでは、ユーザーが Azure ポータルでグループを作成することも、所有者を割り当てることも、有効期限と名前付けポリシーの設定に関する制御を行うことができます。

管理センターでは、所有者以外がゲストを招待できるかどうかを制限するなど、Microsoft 365 管理センターの機能を超えた多くのゲスト招待制御手段も提供されています。

**SharePoint**

SharePoint サイトは、Owner、Member、およびビジタの各セキュリティグループを使用して作成され、最初の2つは対応する Microsoft 365 グループに一致します。 SharePoint Online サイトのメンバーシップは、通常、関連付けられた Microsoft 365 グループによって管理されていますが、双方向リレーションシップではありません。 Microsoft 365 グループレベルのメンバーシップに対する変更は、SharePoint に反映されますが、SharePoint グループでメンバーシップが変更されても、Microsoft 365 グループには反映されません。

### <a name="user-experiences"></a>ユーザーエクスペリエンス

エンドユーザーは、Microsoft 365 内のいくつかのサービスからグループを作成できます。また、他のユーザーはグループとのみ共有できます。

次のサービスを使用すると、エンドユーザーによるグループの作成が可能になります。
                         
Web SharePoint Stream の Outlook Planner プロジェクト Microsoft Teams Yammer

**グループの作成の制限**

チームの増加を制御するための一般的な方法は、ユーザーが作成できるユーザーを制限することです。 これを行うには、グループの作成を制限する必要があります。 これにより、エンドユーザーが必要になる可能性のある他のサービスからグループを作成する機能が影響を受けることになります。 Microsoft 365 グループでは、一部のアプリまたはサービスからのグループの作成を制限する機能をサポートしていません。

グループ作成の制限の効果は、アプリとサービスによって異なります。

## <a name="azure-ad"></a>Azure AD

Azure AD は、Microsoft 365 全体で基盤となる id 管理機能を提供します。

**グループに提供される主な機能**

- グループ メンバーシップ
- 名前付けポリシー
- 有効期限ポリシー
- 者
- グループの作成の制限

**Azure AD でグループを作成できますか?**

はい。 Microsoft 365 グループは、管理 web ポータル、PowerShell、または Graph API を使用して Azure AD から作成できます。

**Azure AD はグループを使用せずに存在しますか?**

はい。 Azure AD は、Microsoft 365 グループと関係のない膨大な数のサービスを実行します。 各 Microsoft 365 グループは、Azure AD のオブジェクトとして表されます。

**グループごとに Azure AD の複数のインスタンスが存在することはできますか?**

いいえ。 Azure AD のインスタンスは1つだけです。

**Azure AD は複数のグループに関連付けることができますか?**

はい。これは、Azure AD がグループメンバーシップサービスを提供する基盤となるプラットフォームであるためです。

**Azure AD とグループの変更を関連付けできますか。**

いいえ。 Azure AD は、グループが存在する基礎となるプラットフォームです。

**インスタンスを削除してグループを削除しますか?**

Azure AD でグループを削除すると、関連するグループ関連のサービスとコンテンツが削除されます。

## <a name="teams"></a>Teams

Teams は、さまざまな Microsoft やサードパーティのサービスと対話するための単一のインターフェイスを提供することによって、共同作業を強化することを目的としたチャット中心のワークスペースです。

既定では、チームを作成すると、Microsoft 365 グループに関連付けられているメールボックスと予定表は、Exchange のグローバルアドレス一覧および Outlook の両方から非表示になります。 ユーザーが同じ Microsoft 365 グループの Outlook と Teams の両方を使用する場合は、管理者が手動で上書きすることができます。

**グループに提供される主な機能**

- 会話
- チャネル & タブ
- 会議

**Teams でグループを作成することはできますか?**

はい。新しいチームを作成すると、新しい Microsoft 365 グループが作成されます。 現在のグループが存在しない既存のグループに対してチームを作成することもできます。

**チームはグループを使用せずに存在しますか?**

いいえ。グループを使用せずにチームを存在させることはできません。

**グループごとに複数のチームが存在することはできますか?**

いいえ。チームとグループの関係は1:1 です。

**チームを複数のグループに関連付けることはできますか?**

いいえ。チーム自体は1つのグループにのみ関連付けることができます。

**チームとグループの間の関連付けは変更できますか。**

いいえ。チームは、もともと関連付けられていたグループにのみ関連付けることができます。

**チームを削除することによってグループを削除するかどうか。**

はい、Microsoft Teams のチームを削除すると、グループ、グループに関連付けられたサービス、およびコンテンツが削除されます。

## <a name="exchange"></a>Exchange

Exchange Online は、メッセージング、予定表、連絡先、および関連付けられた機能を提供します。 グループのコンテキストでは、サービスインスタンス全体ではなく、1つのリソースのみが関連付けられます。

**グループに提供される主な機能**

- メールボックスと予定表
- すべてのグループメンバーにメールを送信する機能
- 電子情報開示のためのチームチャネル会話の保存、Planner コメント

**Exchange でグループを作成できますか。**

はい。 Exchange Online 管理センターおよび Outlook からグループを作成することもできます。 また、Exchange の配布リストを Microsoft 365 グループに変換することもできます。

**Exchange がグループなしで存在するかどうか**

はい。 Exchange Online には、共有されたメールボックスや予定表など、いくつかのサービスがあります。グループの関連付けはありません。

**Exchange メールボックスまたはグループごとのカレンダーのインスタンスが複数存在することはできますか。**

いいえ。グループには、1つの Exchange Online メールボックスと予定表のみを配置できます。

**Exchange メールボックスと予定表は、複数のグループに関連付けることができますか?**

いいえ。メールボックスと予定表には、グループと1:1 の関係があります。 メールボックスを他のユーザーまたはグループと共有することはできますが、これは何らかの形式のサービス関連付けを確立しません。

**Exchange メールボックスまたは予定表がグループとの関連付けを変更できるかどうか。**

いいえ、メールボックスと予定表を別のグループに変更することはできません。 ただし、Outlook 内で、またはサードパーティ製ツールを使用して、あるメールボックスから別のメールボックスにコンテンツを移動することができます。

**メールボックスを削除するグループを削除しますか?**

はい。 Exchange のメールボックスを削除すると、グループに関連付けられたサービスとコンテンツも削除されます。

## <a name="forms"></a>フォーム

フォームには、web ベースのアンケートとクイズが用意されています。

**グループに提供される主な機能**

- フォームの所有権

**フォームでグループを作成することはできますか?**

いいえ、フォームはグループを作成できません。

**フォームはグループ化せずに存在しますか?**

はい、アンケートとクイズをエンドユーザーのアカウントに直接作成することができます。

**グループごとに複数のフォームがあるかどうか**

はい。グループに関連付けられているフォームが複数存在する場合があります。

**フォームは複数のグループに関連付けることができますか?**

いいえ。フォームは1つのグループにのみ関連付けることができます。

**フォームとグループの関連付けを変更できますか。**

フォームがグループ (内部で直接作成されているか、または個別の所有権を持つ) に関連付けられている場合は、別のグループに移動することはできません。

**フォームを削除することによってグループを削除するかどうか。**

いいえ。フォームインターフェイスからグループを削除することはできません。個別のフォームのみです。

## <a name="onenote"></a>OneNote

OneNote はデジタルノートブックアプリケーションです。 グループによって作成された OneNote ノートブックは、グループに接続されたサービスではなく、関連付けられた SharePoint サイトのファイルです。

**グループに提供される主な機能**

- 共有ノートブック (グループに関連付けられた SharePoint ライブラリに保存)

**OneNote はグループを作成できますか?**

いいえ。 OneNote アプリケーションはグループを作成できません。

**OneNote ノートブックはグループを使用せずに存在しますか?**

はい。 OneDrive または他の共有場所にあるノートブックを直接作成することができます。

**グループごとに複数の OneNote ノートブックを使用できますか。**

はい。ノートブックは既定で作成され、他のユーザーを追加できます。ただし、グループに関連付けられたサービスからの OneNote へのリンクは常に既定のノートブックに移動します。

**OneNote ノートブックは複数のグループに関連付けることができますか?**

いいえ。このノートブックは、グループに関連付けられた SharePoint サイトライブラリに格納され、さまざまなインターフェイスからリンクされています。 ただし、個人と共有する場合と同じ方法で、他のグループと共有することができます。

**ノートブックとグループの関連付けは変更できますか。**

いいえ。ノートブック自体はグループに関連付けられており、他のグループに接続されたサービスから直接アクセスすることができますが、コンテンツを OneNote アプリケーション内の別のノートブックに移動できます。

**ノートブックを削除すると、グループが削除されますか。**

ただし、OneNote ノートブックが削除された場合は、一部のグループ関連サービスのリンクが壊れている可能性があります。

## <a name="planner"></a>Planner

Planner は、ライトウェイトグループタスク管理サービスです。

**グループに提供される主な機能**

- グループタスクを管理するためのボード

**プランナーでグループを作成することはできますか?**

はい。プランを作成すると、新しいグループが作成されます。

**Planner ボードはグループを使用せずに存在しますか?**

いいえ。プランは、グループに関連付けられている必要があります。

**グループごとに複数のプランを使用できますか。**

はい。グループごとに複数のプランが存在する場合があります。

**プランは複数のグループに関連付けることができますか?**

いいえ。プランでは、アクセスを決定するためにグループメンバーシップだけに依存しています。

**プランがグループの変更に関連付けられているかどうか。**

いいえ。プランをコピーすると、新しいグループが作成されます。

> [!NOTE]
> その他のアプリケーションによって作成されたグループは、ユーザーに対して自動的に Planner に表示されません。 最初にボードにアクセスするには、Outlook などの別のグループベースのインターフェイスから開く必要があります。

**プランを削除することによってグループを削除するかどうか。**

はい。プランを削除すると、グループとグループに関連付けられたサービスとコンテンツが削除されます。

## <a name="power-apps"></a>Power アプリ

Power Apps には、コードを使用せずにアプリ開発用のキャンバスが用意されています。

**グループに提供される主な機能**

- アプリをグループと共有して、実行および変更を行うことができます。

**電源アプリでグループを作成できますか。**

いいえ。 Power Apps は Microsoft 365 グループを作成できません。

**グループを使用せずに Power Apps が存在するかどうか**

はい。アプリは、共有または公開するまで、Power Apps 内に作成し、クリエーターアカウント内に配置できます。

**グループごとに複数のアプリを使用できますか?**

はい。複数のアプリをグループと共有することができます。

**アプリは複数のグループに関連付けることができますか?**

はい、アプリを複数のグループと共有できます。

**アプリとグループの関連付けを変更できますか。**

はい。電源アプリと Microsoft 365 グループの間の関連付けが共有のみであるため、アプリは作成者と共に存在します。

> [!IMPORTANT]
> [アプリを共有できるようにするには、グループのセキュリティが有効になっている必要があり](https://docs.microsoft.com/powerapps/maker/canvas-apps/share-app#share-an-app-with-office-365-groups)ます。

**アプリを削除すると、グループが削除されますか。**

いいえ。アプリは、そのグループと共有されている以外のグループに接続されていません。

## <a name="power-automate"></a>Power Automate

パワー・オートメーション (旧称 Microsoft Flow) は、ワークフローと自動化サービスを提供します。

**グループに提供される主な機能**

- ワークフローは、グループと共有して、実行および変更することができます。

**電源を自動化してグループを作成できますか?**

いいえ。 Microsoft 365 グループは、そのグループに関連付けられているコンテキストでは作成できません。

ただし、Azure AD セキュリティグループの作成、Microsoft 365 グループのメンバーシップの更新など、さまざまな操作を実行するフローを作成することができます。

**フローがグループ化せずに存在するかどうか**

はい。フローは、電力の自動生成中に作成でき、共有または公開するまでクリエーターアカウント内に配置できます。

**グループごとに複数のフローがありますか。**

はい。グループと複数のフローを共有できます。

**フローは複数のグループに関連付けることができますか?**

はい。複数のグループとフローを共有できます。

**グループの変更に対してフローの関連付けを行うことはできますか。**

はい。電源をオンにした場合と Microsoft 365 グループが共有されている場合の関連付けとして、フローは作成者にも存在します。

**フローを削除してグループを削除するか。**

いいえ。電源アプリと同様に、フローは共有ではないグループに接続されていません。

## <a name="power-bi-classic"></a>Power BI (クラシック)

Power BI は、対話的なデータ駆動型のダッシュボードとレポートを提供します。

**グループに提供される主な機能**

- データの報告

**Power BI がグループを作成できるかどうか。**

はい。クラシックワークスペースを作成すると、Microsoft 365 グループが作成されます。

**Power BI クラシックワークスペースは、グループ化されていませんか?**

いいえ。 [POWER BI のクラシックワークスペースは、グループに関連付けられている必要があり](https://docs.microsoft.com/power-bi/collaborate-share/service-collaborate-power-bi-workspace)ます。

**グループごとに複数の Power BI ワークスペースを使用できますか。**

いいえ。クラシックワークスペースとグループの関係は1:1 です。

**ワークスペースは複数のグループに関連付けることができますか?**

技術的には、クラシックワークスペースはグループと共に作成されますが、ユーザーおよびセキュリティグループを使用して、グループの外部でコンテンツを共有することができます。

**ワークスペースのグループとの関連付けを変更することはできますか。**

いいえ。クラシックワークスペース自体はグループに関連付けられますが、Power BI インターフェイス内のワークスペース間で、またはコンテンツをローカルでエクスポートすることで、コンテンツを別のワークスペースに移動できます。

**ワークスペースを削除するグループを削除しますか?**

はい。 Power BI でワークスペースを削除すると、グループとグループに関連付けられたサービスとコンテンツが削除されます。

## <a name="power-bi-new"></a>Power BI (新機能)

Power BI は、対話的なデータ駆動型のダッシュボードとレポートを提供します。

Power BI で新しいワークスペースを作成しても、Microsoft 365 グループは作成されませんが、その他の方法でグループを作成すると、Power BI に新しい (非クラシック) ワークスペースが作成されます。

**グループに提供される主な機能**

- データの報告

**Power BI がグループを作成できるかどうか。**

いいえ。新しい Power BI インターフェイスから Microsoft 365 グループを作成することはできません。

**新しい Power BI ワークスペースはグループなしで存在しますか?**

はい。 Microsoft 365 グループに関連付けられていないレポートとワークスペースを Power BI で作成することもできます。

**グループごとに複数のワークスペースを使用できますか。**

はい、 [POWER BI によって作成された複数のワークスペースを1つのグループと共有でき](https://docs.microsoft.com/power-bi/collaborate-share/service-create-the-new-workspaces#give-access-to-your-workspace)ます。

**ワークスペースは複数のグループに関連付けることができますか?**

いいえ。 Power BI によって作成されたワークスペースは、1つのグループにのみ関連付けることができます。

**ワークスペースのグループへの関連付けは変更できますか。**

さあ何とも言えません。 Power BI によって作成されたワークスペースは、一度に1つのグループにのみ関連付けることができますが、いつでも関連付けを変更できます。 グループによって Power BI に作成されたワークスペースは、そのグループに永続的に関連付けられます。

**ワークスペースを削除するグループを削除しますか?**

はい。 Power BI でワークスペースを削除すると、グループとグループに関連付けられたサービスとコンテンツが削除されます。

## <a name="project-for-the-web"></a>Web 用 Project

Project for web には、プロジェクト計画、ガントチャート、およびロードマップを作成する機能が用意されています。
グループに提供される主な機能。

- プロジェクト計画

**プロジェクトを web で使用できるグループを作成する**

はい。 web 用の Project から直接新しい Microsoft 365 グループを作成することができます。

**プロジェクトがグループ化せずに存在するかどうか**

はい。 Microsoft 365 グループに関連付けずにプロジェクトを存在させることができます。ただし、タスクの割り当てにはグループの関連付けが必要です。

**グループごとに複数のプロジェクトを使用できますか?**

はい。1つのグループに複数のプロジェクトを接続することもできます。

**プロジェクトを複数のグループに関連付けることはできますか?**

いいえ。プロジェクトは1つのグループにのみ関連付けることができます。

**プロジェクトとグループの間の関連付けは変更できますか。**

いいえ、グループとの関連付けが確立されると、変更できなくなります。

**プロジェクトを削除してグループを削除しますか?**

いいえ。 web のプロジェクトのプロジェクトを削除しても、グループは削除されません。

## <a name="roadmap"></a>ロードマップ

ロードマップでは、project を使用してプロジェクトロードマップを web および Project Online で作成できます。

**グループに提供される主な機能**

- プロジェクトロードマップ

**ロードマップはグループを作成できますか?**

はい。新しい Microsoft 365 グループをロードマップから直接作成することができます。

**ロードマップはグループ化されていませんか?**

はい、Microsoft 365 グループに関連付けられていない状態でロードマップを使用できますが、ロードマップを共有するには、グループの関連付けが必要です。

**グループごとに複数のロードマップを使用できますか。**

はい。複数のロードマップを1つのグループに接続することもできます。

**ロードマップは複数のグループに関連付けることができますか?**

いいえ、ロードマップは1つのグループにのみ関連付けることができます。

**ロードマップがグループの変更と関連付けられているかどうか。**

いいえ、グループとの関連付けが確立されると、変更できなくなります。

**ロードマップを削除するグループを削除しますか?**

いいえ。ロードマップを削除してもグループは削除されません。

## <a name="sharepoint"></a>SharePoint

SharePoint は、さまざまな Microsoft 365 サービスのストレージサービスを提供する web ベースのコンテンツ管理プラットフォームです。

**グループに提供される主な機能**

- ドキュメント ライブラリ
- OneNote ノートブックのストレージ用のライブラリ
- Teams wiki ファイルの保存

**SharePoint でグループを作成できますか?**

はい。 SharePoint でチームサイトを作成すると、既定で Microsoft 365 グループが作成されます。 グループと、必要に応じて、既存のサイトのチームを作成することもできます。

**SharePoint サイトはグループを使用せずに存在しますか?**

はい。 SharePoint には、グループに関連付けられていないいくつかのサービスと、コミュニケーションサイトやハブサイトなどのサイトが用意されています。 

**グループごとに複数のサイトが存在することはできますか?**

いいえ。グループごとに1つのサイトしか存在できません。 Teams のプライベートチャネルは、グループに接続されていない追加サイトを使用します。

**サイトを複数のグループに関連付けることはできますか?**

技術的にはありませんが、グループを使用してサイトを作成すると、そのコンテンツを他のグループと共有できます。

**サイトとグループの関連付けを変更することはできますか。**

いいえ。サイト自体はグループに関連付けられていますが、コンテンツをローカルにエクスポートするか、またはサードパーティ製ツールを使用してサイト間で移動することができます。

**サイトを削除することによってグループを削除するかどうか。**

はい。 SharePoint でサイトを削除すると、グループとグループに関連付けられたサービスとコンテンツが削除されます。

## <a name="stream"></a>Stream

Microsoft Stream は、ビデオをホスティングおよび共有するプラットフォームです。

**グループに提供される主な機能**

- ビデオストレージ
- Teams 会議のレコーディング
- ビデオチャネル

**グループ作成をストリーミングできますか?**

はい。新しい Microsoft 365 グループを Stream から直接作成することができます。

**Stream はグループを使用せずに存在しますか?**

はい、ビデオチャネルとビデオは、グループに関連付けられていないストリームに存在することができます。

**グループごとに複数のビデオやチャネルを使用できますか。**

はい、各グループに複数のビデオやチャネルを配置できます。

**ビデオやチャネルを複数のグループに関連付けることはできますか?**

はい。グループを使用してビデオやチャネルを作成すると、他のグループと共有できます。

**グループとの関連付けを変更できますか。**

はいStream 内のビデオは、元のアップローダーまたは会議レコーダーが所有しているため、任意のグループに関連付けることができます。ただし、ビデオチャネルは、もともと作成されたグループにのみ関連付けることができます。

**ビデオまたはチャネルを削除すると、グループが削除されますか。**

いいえ。ビデオまたはチャネルを削除してもグループは削除されません。 ただし、Stream のグループ自体を削除すると、実際のビデオ以外のグループに関連付けられたサービスとコンテンツが削除されます。

## <a name="yammer"></a>Yammer

Yammer は、組織間および組織間でのコミュニティの参加を促進するために設計されたエンタープライズソーシャルプラットフォームです。

Yammer のコミュニティ (旧称 "グループ") を作成するとメールボックスが作成されますが、現在は使用されていません。

Yammer に関連付けられている Microsoft 365 グループは、Microsoft Teams のチームと一緒に使用することはできません。

**グループに提供される主な機能**

- 会話領域

**Yammer で Microsoft 365 グループを作成することはできますか?**

はい、Yammer で新しいグループを作成すると、新しい Microsoft 365 グループが作成されます (プラットフォームが接続されていて、ユーザーがグループを作成できます)。

Microsoft 365 グループが関連付けられている Yammer グループを、Yammer 自体以外のインターフェイスまたはサービスで作成することはできません。

**Yammer グループは、Microsoft 365 グループなしで存在しますか?**

はい。 Microsoft 365 グループを使用せずに Yammer グループを作成することもできます。

Yammer プラットフォームが Microsoft 365 グループに接続されていない場合や、ユーザーが Microsoft 365 グループを作成できない場合は、Microsoft 365 グループの関連付けなしで Yammer グループが作成されます。

**Microsoft 365 グループあたり複数の Yammer グループが存在することはできますか?**

いいえ、Yammer グループと Microsoft 365 グループの間の関係は1:1 です。

**Yammer グループを複数の Microsoft 365 グループに関連付けることはできますか?**

いいえ。 Yammer グループは、1つの Microsoft 365 グループにのみ関連付けることができます。 他の Yammer グループとの間で投稿を共有したり移動したりすることができます。

**Yammer グループと Microsoft 365 グループの関連付けは変更できますか。**

いいえ。 Yammer グループは、もともと関連付けられていた Microsoft 365 グループにのみ関連付けることができます。

**Yammer グループを削除するには、Microsoft 365 グループを削除しますか?**

はい、Yammer のグループを削除すると、関連する Microsoft グループおよびグループ関連のサービスとコンテンツが削除されます。
