# <a name="gdpr-simplified-a-guide-for-your-small-business"></a>GDPR の簡略化: 小規模ビジネス向けガイド

 *GDPR Microsoft 365の軽減と管理にビジネス向けサービス を使用する* 
  
一般データ保護規則 (GDPR) は、組織が個人データを処理する方法を義務付ける EU (EU) の規制です。 GDPR は 2018 年 5 月 25 日に発効する予定です。 ビジネスが欧州連合 (EU) の市民に対してサービスを販売、提供、または雇用している場合 [、GDPR は](https://ec.europa.eu/info/law/law-topic/data-protection/reform/what-does-general-data-protection-regulation-gdpr-govern_en) ユーザーに影響を与える可能性があります。 
  
この記事では、GDPR の概要、その理由、およびビジネス向Microsoft 365 GDPR の遵守に役立つ方法を理解するのに役立ちます。
  
## <a name="a-quick-overview-of-the-gdpr"></a>GDPR の概要

GDPR は、1995 年に初めて制定された以前のデータ保護ディレクティブ (DPD) を更新および拡張する EU 規制です。 GDPR は、個々のクライアント、顧客、従業員、またはビジネス パートナーなど、個人のデータのプライバシーに関係しています。 GDPR の目標は、EU 市民が EU に居住している場合でも、他の場所に居住している場合でも、EU 市民の個人データ保護を強化することです。 規制は、期待を設定し、それらを達成する方法について助言します。 GDPR が 2018 年 5 月下旬に施行可能になった場合、組織は GDPR の要件を満たす対策を実施する必要があります。
  
GDPR は、データとデータの使い方に関するすべてです。 データはライフ サイクルを持つと考えます。 このサイクルは、データの収集時に開始し、データを保存して使用 (処理) すると続行され、システムから完全に削除されると終了します。 
  
GDPR は、次の種類のデータに関係しています。 
  
- **個人データ:** データを個人にリンクして特定できる場合、そのデータは GDPR に関して個人と見なされます。 個人データの例としては、名前、住所、生年月日、IP アドレスがあります。 GDPR は、データを個人にリンクできる場合、データのあいまい性や技術的な点に関係なく、エンコードされた情報 ("仮名" 情報とも呼ばれる) を個人データと見なします。
    
- **機密性の高い個人データ** これは、個人データに詳細を追加するデータです。 たとえば、宗教、労働組合のメンバーシップ、民族の出身などです。 機密性の高い個人データには、生体認証データと DNA も含まれます。 GDPR では、機密データには個人データよりも厳しい保護ルールがあります。 
    
## <a name="gdpr-terms"></a>GDPR の用語

GDPR で頻繁に参照される用語が表示されます。 これらの用語を理解することが重要です。
  
 **同意**
  
GDPR には、「個人データの処理は、人類にサービスを提供するように設計する必要があります」と示されています。 GDPR は、個人データを処理する際に同意を使用して、この目標を達成する必要があります。 これは、顧客が会社から電子メール メッセージを受信する場合に顧客に尋ねる簡単な行為です。 また、マーケティングにデータを使用する場合は、Web サイトでオプトアウト チェック ボックスを追加する必要はありません。 「明確な肯定的な行為」を使用して明示的な同意を得る必要があります。 また、同意が取り消された場合の記録も保持する必要があります。
  
 **データ主体の権限**
  
GDPR はデータ主体の権利を確立します。つまり、個人データに関して、顧客、従業員、ビジネス パートナー、クライアント、請負業者、学生、サプライヤーなどには、次の権利があります。
  
- **データに関する情報を受け取る:** データの使用について個人に通知する必要があります。 
    
- **データへのアクセス権を持つ:** 保持しているデータ (たとえば、アカウント アクセスを使用したり、手動で) にアクセスしたりするには、個人にアクセス権を与える必要があります。 
    
- **データの変更を求める:** 個人は不正確なデータの修正を求める場合があります。 
    
- **データの削除を求める:** 「消去の権利」とも呼ばれるこの権利により、個人は、企業が収集した個人データを、それを使用または共有しているすべてのシステムで削除を要求できます。 
    
- **制限付き処理を要求する:** 個人は、データを抑制または制限する必要があります。 ただし、特定の状況でのみ適用されます。 
    
- **データの移植性を持つ:** 個人は、自分のデータを別の会社に転送する必要があります。 
    
- **オブジェクト:** 個人は、ダイレクト マーケティングを含むさまざまな用途で使用されるデータに対して、オブジェクトを作成できます。 
    
- **プロファイリングを含む、自動化された意思決定の対象になさらなさる。** GDPR には、データを使用してユーザーをプロファイルし、そのプロファイルに基づいて意思決定を自動化する厳密なルールがあります。 
    
## <a name="common-small-business-questions-about-the-gdpr"></a>GDPR に関する一般的な小規模ビジネスの質問

### <a name="im-a-sole-proprietor---do-i-really-have-to-worry-about-the-gdpr"></a>私は個人事業主です - 本当に GDPR について心配する必要がありますか?

GDPR は、処理するデータに関する情報で、従業員の数ではありません。 これは、個人事業主でさえ、すべてのサイズの企業に影響を与える。 ただし、従業員数が 250 未満の企業では、記録保持の削減など、一部の除外措置が適用されますが、データ処理が個人の権利に影響を与えず、時折処理される場合に限ります。
  
例として、個人以外のデータの処理は免除されるか、または減らされた措置が必要になります。 ただし、"特殊なカテゴリ機密データ" と見なされるデータを処理する場合、場合によってはデータ処理を記録する必要があります。 "たまに処理" の定義はあいまいですが、一度またはまれに使用されるデータに適用することを意図しています。
  
また、収集する個人データが保護されている必要があります。 つまり、暗号化して、少なくともパスワードを使用してアクセスを制御する必要があります。 顧客データをデスクトップ上のスプレッドシートに保存しても、保護は GDPR の期待を満たされません。 
  
### <a name="how-can-i-tell-if-our-company-website-is-gdpr-compliant"></a>会社の Web サイトが GDPR に準拠している場合は、どうすれば確認できますか?

最初に質問する質問は、サイトの任意の場所で個人データを収集するかです。 たとえば、名前と電子メール アドレスを求める連絡先フォームがある場合があります。 マーケティング メールを送信する場合は、データを使用する内容を正確に説明する 「オプトイン」チェック ボックスを必ず追加してください。 受信者がチェックした場合にのみ、マーケティング目的で自分の個人データを使用できます。
  
また、データを格納するデータベースが保護されているのを確認します。 Web ホスティング会社またはクラウド ストレージ ベンダーは、この情報を提供できます。 ビジネス用にMicrosoft 365を使用する場合、データの保存は GDPR に準拠しています。 
  
### <a name="my-company-is-outside-europe-does-the-gdpr-really-affect-us"></a>私の会社はヨーロッパ以外です。 GDPR は実際に影響しますか?

GDPR は、EU 市民を保護する規制です。 現在、会社が EU 市民を扱っている場合、または将来の希望がある場合は、影響を受けます。 これは、EU 加盟国に住む市民と他の地域に住む市民の両方に適用されます。
  
以下の場合を例とします。
  
- EU 市民に車を雇う米国の企業は、顧客のデータを収集および処理する際に GDPR 要件を満たす必要があります。 会社は、お客様のデータを取得する際に同意を得て、データが安全に保存される必要があります。 また、顧客がすべてのデータ主体権限を適用できる必要があります。 
    
- オーストラリアの会社がオンラインで製品を販売し、ユーザーがオンライン アカウントを設定します。 GDPR データ主体の権利と同意は、アカウントを開く EU 市民に適用されます。 会社は、顧客がすべてのデータ主体権限を適用できる必要があります。 
    
- 国際的な慈善団体は、寄付者に関するデータを収集し、それを使用して、寄付の更新や要求を送信します。 GDPR の状態は '... です。直接マーケティング目的での個人データの処理は、正当な利益のために行われると見なされる可能性があります。 ただし、組織は、自分の関心がデータ主体の利益を上書きする責任を負います。 会社 (またはこの場合は、慈善団体) は、常に通知、明示的、オプトインの同意を得る必要があります。
    
GDPR は、顧客データが国境を越えて移動する場合にも適用されます。 データ ストレージにクラウド コンピューティングを使用する場合は、サービスが GDPR に完全に準拠しているか確認する必要があります。 データストレージがデータ保護の記録が不十分な場所にある場合は、複雑になる可能性があります。 ビジネス向けMicrosoft 365場合、GDPR 要件をカバーする適切な法的文書が用意されています。
  
### <a name="sure-i-collect-data-but-some-other-company-stores-it-does-that-get-me-off-the-hook"></a>確かに、私はデータを収集しますが、他の会社がデータを保存します。 それは私をフックから離れるのですか?

GDPR では、データを収集すると、ある程度影響を受ける可能性があります。 GDPR には、データ プロセッサとデータ コントローラーの概念があります。 
  
- **データ コントローラー:** データの収集方法、処理方法、および収集理由を決定する個人または組織 (共同管理者を持つ場合があります)。 他の会社のクラウド サーバーを使用して保存する場合があります。 たとえば、顧客データを収集する Web サイトはコントローラーです。 
    
- **データ プロセッサ:** 管理者に代わってデータを格納し、要求に応じてこれらのデータを処理する個人または組織。 たとえば、データ ストレージMicrosoft 365 Apps for businessプロセッサとして機能し、GDPR に完全に準拠しています。 
    
    組織またはシステムは、コントローラーとプロセッサの両方として機能できます。 Microsoft 365は、GDPR の両方として機能し、準拠できます。
    
### <a name="can-i-still-send-out-marketing-emails-to-my-old-customers"></a>古い顧客にマーケティングメールを送信できますか?

顧客 (何年も持っていた顧客も含む) が、マーケティングにデータを使用する同意を得たことを確認する必要があります。 以前に同意を取得した可能性があります。また、それを表示するレコードも含む場合があります。 その場合は、マーケティングを続行する必要があります。 ない場合は、顧客にマーケティングを続ける権限を顧客から取得する必要があります。 これは通常、顧客にサイトに移動し、将来のメールを受信する同意するオプションを選択する電子メールを送信する必要があります。 
  
### <a name="do-i-have-to-worry-about-the-gdpr-when-i-recruit-new-employees-what-about-current-employees"></a>新入社員を募集するときに GDPR について心配する必要がありますか? 現在の従業員について

GDPR は、お客様のデータに影響を与えるだけという意味で機能します。従業員データにも拡張されます。 多くの場合、新入社員は LinkedIn などのソーシャル メディア プラットフォームを使用して見つけ入れる。 明示的な許可なしに、潜在的な採用データを保存しなかからなくしてください。
  
既存の従業員と新入社員契約に関して、契約の最後の署名は、特に契約で肯定的でない条項が使用されている場合、必ずしも同意を前提としません。 この場合、条項に関連付けられた明示的な方法で同意を取得する必要があります。 これは、従業員契約によって異なりますが、場合によっては「正当な利益」を使用し、従業員のデータ処理通知を追加して、従業員が自分のデータに対して何を行うのかを確認できます。
  
## <a name="satisfy-privacy-concerns-using-microsoft-365-for-business"></a>ビジネス向けサービスを使用してMicrosoft 365問題を満たす

GDPR に準拠するには、個人データが確実に保護されます。 GDPR には、デザインと既定によるプライバシーと呼ばれる概念があります。 つまり、データ保護はシステムと製品に対して "ベイクイン" する必要があります。プライバシーに関する懸念を満たすのは第 2 の性質です。 
  
小規模ビジネスでは、大規模なビジネスと同様に、セキュリティを犠牲にすることなく利便性が必要です。 Microsoft 365は、従業員数が 300 人未満の企業向けです。 小規模な企業は、Microsoft クラウドベースのツールを使用してビジネスの生産性を向上させることができます。 ビジネスMicrosoft 365、小規模企業はメール、ドキュメント、会議やイベントを管理できます。 また、GDPR コンプライアンスに不可欠な組み込みのセキュリティ対策とデバイス管理も備え付けされています。
  
Microsoft 365は、GDPR プロセスを以下の方法で支援します。
  
- **検出:** GDPR コンプライアンスの重要なステップは、ユーザーが持っているデータを知ることです。 
    
- **管理:** データへのアクセスを制御し、その使用を管理する方法は、GDPR の不可欠な部分です。 Microsoft 365の場合、デバイスに適用するポリシーに基づいてビジネス データが保護されます。 デバイス管理は、従業員がリモートで作業する時代に不可欠です。 Microsoft 365には、すべてのデバイスでデータが保護されるデバイス管理機能が含まれています。 たとえば、ビジネス内のすべてのデバイスをWindows 10デバイスを使用して保護Windows Defender。 
    
- **保護: Microsoft 365** セキュリティを目的として設計されています。 デバイス管理とデータ保護制御は、リモート デバイスを含むビジネス ネットワーク全体で機能し、データのセキュリティ保護を支援します。 Microsoft 365アプリケーションのプライバシー設定やドキュメントの暗号化などのOfficeを提供しています。 ビジネスMicrosoft 365を使用すると、GDPR コンプライアンス監視を実行して、適切なレベルの保護セットを確実に設定できます。 
    
- **レポート:** GDPR はレポートに重点を置いて行います。 従業員が 1 人のビジネスでも、大量のデータを処理する場合は、その手順を文書化して報告する必要があります。 Microsoft 365は、小規模な組織の報告要件から頭痛の種を取り除く。 
    
    監査ログなどのツールを使用すると、データの移動を追跡およびレポートできます。 レポートには、収集および保存するデータの分類、データの処理、データの転送が含まれます。 
    
顧客、従業員、およびクライアントは、データプライバシーの重要性を認識し、企業や組織がプライバシーを尊重する必要があります。 Microsoft 365は、ビジネスに大きな動きを生じることなく、GDPR コンプライアンスを達成し維持するためのツールを提供します。
