<span data-ttu-id="60a82-115">GDPR は、データとデータの使い方に関するすべてです。</span><span class="sxs-lookup"><span data-stu-id="60a82-115">The GDPR is all about data and how it's used.</span></span> <span data-ttu-id="60a82-116">データはライフ サイクルを持つと考えます。</span><span class="sxs-lookup"><span data-stu-id="60a82-116">Think of data as having a life cycle.</span></span> <span data-ttu-id="60a82-117">このサイクルは、データの収集時に開始し、データを保存して使用 (処理) すると続行され、システムから完全に削除されると終了します。</span><span class="sxs-lookup"><span data-stu-id="60a82-117">The cycle starts when you collect data, continues as you store it and use it (processing), and ends when you completely delete it from your systems.</span></span> 
  
<span data-ttu-id="60a82-118">GDPR は、次の種類のデータに関係しています。</span><span class="sxs-lookup"><span data-stu-id="60a82-118">The GDPR is concerned with the following types of data:</span></span> 
  
- <span data-ttu-id="60a82-119">**個人データ:** データを個人にリンクして特定できる場合、そのデータは GDPR に関して個人と見なされます。</span><span class="sxs-lookup"><span data-stu-id="60a82-119">**Personal data:** If you can link data to an individual and identify them, then that data is considered personal with respect to the GDPR.</span></span> <span data-ttu-id="60a82-120">個人データの例としては、名前、住所、生年月日、IP アドレスがあります。</span><span class="sxs-lookup"><span data-stu-id="60a82-120">Examples of personal data include name, address, date of birth, and IP address.</span></span> <span data-ttu-id="60a82-121">GDPR は、データを個人にリンクできる場合、データのあいまい性や技術的な点に関係なく、エンコードされた情報 ("仮名" 情報とも呼ばれる) を個人データと見なします。</span><span class="sxs-lookup"><span data-stu-id="60a82-121">The GDPR considers even encoded information (also known as "pseudonymous" information) to be personal data, regardless of how obscure or technical the data is, if the data can be linked to an individual.</span></span>
    
- <span data-ttu-id="60a82-122">**機密性の高い個人データ** これは、個人データに詳細を追加するデータです。</span><span class="sxs-lookup"><span data-stu-id="60a82-122">**Sensitive personal data** This is data that adds more details to personal data.</span></span> <span data-ttu-id="60a82-123">たとえば、宗教、労働組合のメンバーシップ、民族の出身などです。</span><span class="sxs-lookup"><span data-stu-id="60a82-123">Examples include religion, trade union membership, ethnic origin, and so on.</span></span> <span data-ttu-id="60a82-124">機密性の高い個人データには、生体認証データと DNA も含まれます。</span><span class="sxs-lookup"><span data-stu-id="60a82-124">Sensitive personal data also includes biometric data and DNA.</span></span> <span data-ttu-id="60a82-125">GDPR では、機密データには個人データよりも厳しい保護ルールがあります。</span><span class="sxs-lookup"><span data-stu-id="60a82-125">Under GDPR, sensitive data has more stringent protection rules than personal data.</span></span> 
    
## <a name="gdpr-terms"></a><span data-ttu-id="60a82-126">GDPR の用語</span><span class="sxs-lookup"><span data-stu-id="60a82-126">GDPR terms</span></span>

<span data-ttu-id="60a82-127">GDPR で頻繁に参照される用語が表示されます。</span><span class="sxs-lookup"><span data-stu-id="60a82-127">You'll see some terms referred to frequently in the GDPR.</span></span> <span data-ttu-id="60a82-128">これらの用語を理解することが重要です。</span><span class="sxs-lookup"><span data-stu-id="60a82-128">It's important to understand these terms.</span></span>
  
 <span data-ttu-id="60a82-129">**同意**</span><span class="sxs-lookup"><span data-stu-id="60a82-129">**Consent**</span></span>
  
<span data-ttu-id="60a82-130">GDPR には、「個人データの処理は、人類にサービスを提供するように設計する必要があります」と示されています。</span><span class="sxs-lookup"><span data-stu-id="60a82-130">The GDPR states: "The processing of personal data should be designed to serve mankind."</span></span> <span data-ttu-id="60a82-131">GDPR は、個人データを処理する際に同意を使用して、この目標を達成する必要があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-131">The GDPR hopes to achieve this goal by using consent when processing personal data.</span></span> <span data-ttu-id="60a82-132">これは、顧客が会社から電子メール メッセージを受信する場合に顧客に尋ねる簡単な行為です。</span><span class="sxs-lookup"><span data-stu-id="60a82-132">That could be the simple act of asking your customers if they want to receive email messages from your company.</span></span> <span data-ttu-id="60a82-133">また、マーケティングにデータを使用する場合は、Web サイトでオプトアウト チェック ボックスを追加する必要はありません。</span><span class="sxs-lookup"><span data-stu-id="60a82-133">It also means no more opt-out check boxes on your website when you want to use data for marketing.</span></span> <span data-ttu-id="60a82-134">「明確な肯定的な行為」を使用して明示的な同意を得る必要があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-134">You must take explicit consent using a "clear affirmative act".</span></span> <span data-ttu-id="60a82-135">また、同意が取り消された場合の記録も保持する必要があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-135">And, you will need to also keep records of when a consent is taken or revoked.</span></span>
  
 <span data-ttu-id="60a82-136">**データ主体の権利**</span><span class="sxs-lookup"><span data-stu-id="60a82-136">**Data subject rights**</span></span>
  
<span data-ttu-id="60a82-137">GDPR はデータ主体の権利を確立します。つまり、個人データに関して、顧客、従業員、ビジネス パートナー、クライアント、請負業者、学生、サプライヤーなどには、次の権利があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-137">The GDPR establishes data subject rights, which means that, with respect to their personal data, customers, employees, business partners, clients, contractors, students, suppliers, and so forth have the right to:</span></span>
  
- <span data-ttu-id="60a82-138">**データに関する情報を受け取る:** データの使用について個人に通知する必要があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-138">**Be informed about their data:** You must inform individuals about your use of their data.</span></span> 
    
- <span data-ttu-id="60a82-139">**データへのアクセス権を持つ:** 保持しているデータ (たとえば、アカウント アクセスを使用したり、手動で) にアクセスしたりするには、個人にアクセス権を与える必要があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-139">**Have access to their data:** You must give individuals access to any of their data that you hold (for example, by using account access or in some manual manner).</span></span> 
    
- <span data-ttu-id="60a82-140">**データの変更を求める:** 個人は不正確なデータの修正を求める場合があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-140">**Ask for data rectification:** Individuals can ask you to correct inaccurate data.</span></span> 
    
- <span data-ttu-id="60a82-141">**データの削除を求める:** 「消去の権利」とも呼ばれるこの権利により、個人は、企業が収集した個人データを、それを使用または共有しているすべてのシステムで削除を要求できます。</span><span class="sxs-lookup"><span data-stu-id="60a82-141">**Ask for data to be deleted:** Also known as the 'right to erasure', this right allows an individual to request that any of their personal data a company has collected is deleted across all systems that use it or share it.</span></span> 
    
- <span data-ttu-id="60a82-142">**制限付き処理を要求する:** 個人は、データを抑制または制限する必要があります。</span><span class="sxs-lookup"><span data-stu-id="60a82-142">**Request restricted processing:** An individual can ask that you suppress or restrict their data.</span></span> <span data-ttu-id="60a82-143">ただし、特定の状況でのみ適用されます。</span><span class="sxs-lookup"><span data-stu-id="60a82-143">However, it is only applicable under certain circumstances.</span></span>

- <span data-ttu-id="60a82-145">**オブジェクト:** 個人は、ダイレクト マーケティングを含むさまざまな用途で使用されるデータに対して、オブジェクトを作成できます。</span><span class="sxs-lookup"><span data-stu-id="60a82-145">**Object:** An individual can object to their data being used for various uses including direct marketing.</span></span>
