# <a name="get-started-with-compliance-manager"></a>コンプライアンス マネージャーの使用を開始する

**この記事では、次の情報を参照してください。** この記事は、コンプライアンス マネージャーのセットアップに役立ちます。 コンプライアンス マネージャーにアクセス **し、** 役割 **とアクセス許可を設定** し、改善アクションの自動テストを **構成する方法について学習します**。 コンプライアンス マネージャー **ダッシュボードを参照** し、メイン ページ ([改善アクション] ページ、ソリューション ページ、評価ページ、評価テンプレート ページ) について理解します。

## <a name="who-can-access-compliance-manager"></a>コンプライアンス マネージャーにアクセスできるユーザー

コンプライアンス マネージャーは、Office 365 ライセンスと Microsoft 365 ライセンスを持つ組織、および米国政府機関コミュニティ クラウド (GCC) モデレートおよび GCC High のお客様に利用できます。 評価の可用性と管理機能は、ライセンス契約によって異なっています。  [サービスの説明の詳細を表示します](https://go.microsoft.com/fwlink/?linkid=2132371)。

## <a name="before-you-begin"></a>はじめに

組織の Microsoft 365 グローバル管理者が、コンプライアンス マネージャーにアクセスする最初のユーザーになる可能性があります。 コンプライアンス マネージャーに初めてアクセスする場合は、グローバル管理者がサインインし、以下に示すユーザーアクセス許可を設定することをお勧めします。

## <a name="sign-in"></a>サインイン

1. [Microsoft 365 コンプライアンス](https://compliance.microsoft.com/)センターに移動し **、Microsoft** 365 グローバル管理者アカウントでサインインします。
2. 左側 **のナビゲーション ウィンドウで** [コンプライアンス マネージャー] を選択します。 コンプライアンス マネージャー ダッシュボードに [到着します](#understand-the-compliance-manager-dashboard)。

コンプライアンス マネージャーにアクセスする直接リンクはです [https://compliance.microsoft.com/compliancemanager](https://compliance.microsoft.com/compliancemanager) 。

## <a name="set-user-permissions-and-assign-roles"></a>ユーザーのアクセス許可を設定し、役割を割り当てる

コンプライアンス マネージャーは、役割ベースのアクセス制御 (RBAC) アクセス許可モデルを使用します。 役割が割り当てられているユーザーだけがコンプライアンス マネージャーにアクセスできます。各ユーザーが許可するアクションは、役割の種類によって [制限されます](#role-types)。

### <a name="where-to-set-permissions"></a>アクセス許可を設定する場所

組織のグローバル管理者の役割を持つユーザーは、コンプライアンス マネージャーのユーザーアクセス許可を設定できます。 アクセス許可は、Office 365 セキュリティ &コンプライアンス センターと Azure Active Directory (Azure Active Directory) で設定AD。

> [!NOTE]
> 米国政府機関 (GCC) の高い環境のお客様は、Azure AD のコンプライアンス マネージャーに対するユーザーのアクセス許可と役割のみを設定AD。 Azure の詳細な手順ADロールの種類の定義については、以下を参照してください。

コンプライアンス センターでアクセス許可を設定し、Office 365 セキュリティ &役割を割り当てるには、次の手順を実行します。

1. コンプライアンス センターの[Office 365 セキュリティ &に移動](https://protection.office.com/)し、左側のナビゲーションで [アクセス許可] を選択します。

2. 1 人または複数のユーザーを追加する役割グループを見つけて、グループ名の左側にあるボックスをオンにします。 (以下の役割 [と関連する機能の一覧を参照してください](#role-types)。 役割グループ名は、役割名を模倣します)。

3. そのグループのフライアウト ウィンドウで、[メンバー] ヘッダー **の下にある [編集** ] **を選択** します。

4. [メンバー **の選択] を選択します**。 別のフライアウト ウィンドウが表示されます。

5. [+ **追加] を** 選択して、グループに追加する 1 つ以上のユーザーを選択します。

6. 追加する名前の横にあるチェック ボックスをオンにし、下部にある **[追加** ] ボタンを選択します。

7. ユーザーの割り当てが完了したら、[完了] を選択し、[保存] 、[**閉** じる] の順に **選択します**。

## <a name="settings-for-automated-testing-and-user-history"></a>自動テストとユーザー履歴の設定

Microsoft 365 コンプライアンス センターのコンプライアンス マネージャー設定を使用すると、改善アクションの自動テストを有効または無効にできます。 また、この設定を使用すると、改善アクションに関連付けられたユーザーのデータ (改善アクションを別のユーザーに再割り当てする機能など) を管理できます。  コンプライアンス マネージャーの設定にアクセスできるのは、グローバル管理者またはコンプライアンス マネージャー管理者の役割を持つユーザーのみです。

> [!NOTE]
> これらの環境では Secure Score を使用できないので、GCC High 環境のユーザーは自動テスト機能を使用できません。 GCC High のお客様は、改善アクションを手動で実装してテストする必要があります。

### <a name="set-up-automated-testing"></a>自動テストのセットアップ

コンプライアンス マネージャーの一部の改善アクションも [Microsoft Secure Score によって監視されます](https://docs.microsoft.com/microsoft-365/security/mtp/microsoft-secure-score)。 共同で監視されるアクションの自動テストを設定できます。つまり、セキュリティで保護されたスコアでアクションがテストおよび更新された場合、それらの結果はコンプライアンス マネージャーの同じアクションと同期し、コンプライアンス スコアにカウントされます。

コンプライアンス マネージャーが新しい組織では、既定で自動テストが有効になっています。 Microsoft 365 または Office 365 を初めて展開する場合、Secure Score がデータを完全に収集し、コンプライアンス スコアに組み込むには約 7 日かかる。  自動テストを有効にすると、アクションのテスト日は更新されませんが、テストの状態は更新されます。 新しい評価が作成されると、スコアには Microsoft コントロール スコアと Secure Score 統合が自動的に含まれます。

組織のグローバル管理者は、自動テストの設定をいつでも変更できます。 一般的な改善アクションの自動テストをオフにしたり、個々のアクションに対して有効にしたりできます。 自動テスト設定を変更するには、以下の手順に従います。

#### <a name="to-manage-your-automated-testing-settings"></a>自動テスト設定を管理するには、次の操作を行います。

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. [設定] ページで、[コンプライアンス マネージャー] **を選択します**。

3. 左側 **のナビゲーションから [自動** テスト] を選択します。

4. 該当するボタンを選択して、すべての改善アクションの自動テストを有効にするか、すべてのアクションでオフにするか、個別のアクションでオンにします。

5. [改善アクションごとに **有効にする] を選択すると**、選択できるすべての改善アクションがリストに表示されます。  自動的にテストするアクションの横にあるチェック ボックスをオンにします。

6. [保存 **] を** 選択して設定を保存します。 選択内容が保存されたという確認メッセージが画面の上部に表示されます。 エラー通知を受け取った場合は、もう一度お試しください。

**注:** すべてのアクションに対して自動更新を有効またはオフにできるのは、グローバル管理者のみです。 コンプライアンス マネージャー管理者は、個々のアクションの自動更新を有効にできますが、すべてのアクションに対してグローバルに有効にできるのではありません。

### <a name="manage-user-history"></a>ユーザー履歴の管理

[ **ユーザー履歴の管理]** 設定を使用すると、コンプライアンス マネージャーで改善アクションを実行したユーザーをすばやく特定できます。 改善アクションに関連付けられた識別可能なユーザー データには、実装とテストの作業、アップロードしたドキュメント、および入力したメモが含まれます。 この種類のデータの理解と取得は、組織独自のコンプライアンスニーズに必要な場合があります。

また、ユーザー履歴設定を使用すると、すべての改善アクションをあるユーザーから別のユーザーに再割り当てすることもできます。

**ユーザー履歴の設定を見つけるには、次の方法を実行します。**

1. [Microsoft 365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. [設定] ページで、[コンプライアンス マネージャー] **を選択します**。

3. 左側の **ナビゲーションから [ユーザー履歴の** 管理] を選択します。

[ **ユーザー履歴の管理]** ページには、改善アクションに割り当てられている電子メール アドレス別のすべてのユーザーの一覧が表示されます。 [検索 **] ボタンを** 使用して、メール アドレスを入力して特定のユーザーをすばやく検索します。

各ユーザーのメール アドレスの右側にある [選択] ドロップダウン メニューには、レポートのエクスポート、改善アクションの再割り当て、履歴の削除を行うオプションが用意されています。 各オプションの詳細については、以下の各セクションを参照してください。

#### <a name="export-a-report-of-user-history-data"></a>ユーザー履歴データのレポートをエクスポートする

ユーザーに現在割り当てられている改善アクションの一覧を含む Excel ファイルをエクスポートできます。  このレポートには、そのユーザーがアップロードした証拠ファイルも一覧表示されます。 この情報は、オープン改善アクションを再割り当てするのに役立ちます。

レポートには、作成日の改善アクションの状態が反映されます。 これは、そのステータスまたは割り当てに対する以前のすべての変更の履歴レポートではありません (改善アクション ページからレポートをエクスポートする方法 [について説明します](compliance-manager-improvement-actions.md#export-a-report))。

**以下の手順に従って、ユーザー別にレポートをエクスポートします。**

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. [設定] ページで、[コンプライアンス マネージャー] **を選択します**。

3. 左側の **ナビゲーションから [ユーザー履歴** の管理] を選択します。

4. リストの電子メール アドレスを検索するか、[検索] を選択してユーザーの電子メール アドレスを入力して、目的のユーザーを検索します。

5. [選択 **] ドロップダウン メニュー** から、[レポートのエクスポート] **を選択します**。

6. レポートの Excel ファイルが生成されると、それを開いてローカル コンピューターに保存できます。

#### <a name="reassign-improvement-actions-to-another-user"></a>改善アクションを別のユーザーに再割り当てする

改善アクションは、あるユーザーから別のユーザーに再割り当てできます。 アクションを再割り当てしても、ドキュメントのアップロード履歴は変更されませんが、最初にドキュメントをアップロードしたユーザーの名前は、改善アクション内に表示されなくなりました。

**次の手順に従って、改善アクションを別のユーザーに再割り当てします。**

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. [設定] ページで、[コンプライアンス マネージャー] **を選択します**。

3. 左側の **ナビゲーションから [ユーザー履歴** の管理] を選択します。

4. リストの電子メール アドレスを検索するか、[検索] を選択してそのユーザーの電子メール アドレスを入力して、ユーザーを検索します。

5. [選択] **ドロップダウン メニュー** から、[改善アクションの再割 **り当て] を選択します**。 [ **改善アクションの再割り当て]** フライアウト ウィンドウが表示されます。

6. [ユーザー **の検索]** フィールドに、改善アクションを割り当てるユーザーの名前または電子メール アドレスを入力 *します*。

7. [改善アクションが割り当てられる]の下に目的のユーザーの名前が表示された場合は、ユーザーを選択し、[アクションの割り当て]**を選択します**。

8. 再割り当てが完了すると、フライアウト ウィンドウに確認メッセージが表示され、前のユーザーのすべての改善アクションが新しいユーザーに再割り当てされたという確認メッセージが表示されます。 再割り当てエラー通知を受け取った場合は、ウィンドウを閉じてもう一度やり直してください。 フライアウト ウィンドウを閉じるには、[完了] を **選択します**。

新しい割り当て先は、改善アクションに割り当てられた電子メールを受信します。 電子メールには、改善アクションの詳細ページへの直接リンクが含まれている。
 
 > [!NOTE]
> 保留中の更新があるアクションを再割り当てすると、再割り当て後に更新が受け入れられると、再割り当てメール内のアクションへの直接リンクが壊れる。 更新プログラムが受け入れられると、ユーザーにアクションを再割り当てすることで、この問題を解決できます。 改善アクションの [更新について詳しくは、以下を参照してください](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)。

#### <a name="delete-user-history"></a>ユーザー履歴の削除

ユーザーの履歴を削除すると、ユーザーは改善アクションの所有者として削除され、コンプライアンス マネージャーの他のすべてのフィールドから名前が削除されます。 ユーザーの履歴を削除すると、新しいユーザーが割り当てられるまで、ユーザーが所有する改善アクションには [割り当て済み] 値は表示されません。 改善アクションにアップロードされたドキュメントには、 **削除されたユーザー** の名前の代りで削除されたユーザーが表示されます。 ユーザー履歴の削除は永続的です。

ユーザーの履歴を削除するには、次の手順に従います。

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. [設定] ページで、[コンプライアンス マネージャー] **を選択します**。

3. 左側の **ナビゲーションから [ユーザー履歴** の管理] を選択します。

4. リストの電子メール アドレスを検索するか、[検索] を選択してそのユーザーの電子メール アドレスを入力して、ユーザーを検索します。

5. [選択 **] ドロップダウン メニュー** から、[履歴の削除] **を選択します**。

6. ユーザーの履歴の完全な削除を確認するウィンドウが表示されます。 削除を続行するには、[履歴の削除 **] を選択します**。 履歴を削除せずに残す場合は、[キャンセル] を **選択します**。

7. [ユーザー履歴の管理]ページに戻り、ユーザーの履歴が削除されたという確認メッセージが上部に表示されます。

## <a name="understand-the-compliance-manager-dashboard"></a>コンプライアンス マネージャー ダッシュボードについて

コンプライアンス マネージャー ダッシュボードは、現在のコンプライアンス体制を一目で確認できる設計です。

![コンプライアンス マネージャー - ダッシュボード](../media/compliance-manager-dashboard.png "コンプライアンス マネージャー ダッシュボード")

### <a name="overall-compliance-score"></a>全体的なコンプライアンス スコア

コンプライアンス スコアがトップに目立つように表示されます。 重要なデータ保護の基準と規制に対応する改善アクションを完了するために達成可能なポイントに基づいてパーセンテージを示します。 Microsoft の [アクション (Microsoft](compliance-manager-assessments.md#microsoft-actions-tab)が管理する) のポイントも、コンプライアンス スコアにカウントされます。

コンプライアンス マネージャーに初めてアクセスすると、最初のスコアは [Microsoft 365 データ保護基準に基づいて決されます](compliance-manager-assessments.md#data-protection-baseline-default-assessment)。 このベースライン評価は、すべての組織で利用できます。これは、一般的な業界の規制と標準を含む一連のコントロールです。 コンプライアンス マネージャーは、既存の Microsoft 365 ソリューションをスキャンし、現在のプライバシーとセキュリティ設定に基づいて初期評価を行います。 組織に関連する評価を追加すると、スコアの意味が高くなります。

**詳細: コンプライアンス**[スコアの計算方法について説明します](compliance-score-calculation.md)。

### <a name="key-improvement-actions"></a>主な改善アクション

このセクションでは、コンプライアンス の全体的なスコアに最も大きなプラスの影響を与える場合に、今実行できるトップの改善アクションを示します。 [すべての **改善アクションを表示する] を** 選択して、改善アクション ページに移動します。

### <a name="solutions-that-affect-your-score"></a>スコアに影響を与えるソリューション

このセクションでは、スコアにプラスの影響を与える可能性のある改善アクションと、それらのソリューションの優れた改善アクションの数を含むソリューションについて説明します。 [すべての **ソリューションを表示する] を** 選択して、ソリューション ページにアクセスします。

### <a name="compliance-score-breakdown"></a>コンプライアンス スコアの内訳

このセクションでは、2 つの異なる方法でスコアの詳細なビューを提供します。

- **カテゴリ**: データ保護カテゴリ内の全体的なスコアの割合 (「情報の保護」や「デバイスの管理」など) を示します。
- **評価**: GDPR や NIST 800-53 などの特定のコンプライアンスおよびデータ保護基準、規制、または法律に対する評価の管理における進捗状況の割合を示します。

### <a name="filtering-your-dashboard-view"></a>ダッシュボード ビューのフィルター処理

ダッシュボード ビューをフィルター処理して、特定の規制と標準、ソリューション、アクションの種類、評価グループ、またはデータ保護カテゴリに関連するアイテムのみを表示できます。 この方法でビューをフィルター処理すると、ダッシュボードのスコアもフィルター処理され、フィルター条件に基づいて、可能な合計ポイントから達成したポイント数が表示されます。

フィルターを適用するには、次の方法を使用します。

1. ダッシュボード **の右上隅** にある [フィルター] を選択します。
2. [フィルター] フライアウト ウィンドウからフィルター **条件** を選択し、[適用] を **選択します**。

フィルターを適用すると、リアルタイムでスコアが調整されます。 コンプライアンス スコアの割合と内訳情報、および改善アクションとソリューションは、フィルター条件でカバーされるデータにのみ関連しています。 コンプライアンス マネージャーからサインアウトすると、サインイン時にフィルター処理されたビューが残ります。

フィルターを削除するには、次の方法を使用します。

- コンプライアンス スコア **の上にある** [適用されたフィルター] 見出しで、削除する個々のフィルターの横にある **[X]** を選択します。または
- ダッシュボード **の右上隅** にある [フィルター] を選択し、[フィルター]フライアウト ウィンドウで 、[フィルターのクリア]**を選択します**。

## <a name="improvement-actions-page"></a>[改善アクション] ページ

[改善アクションは](compliance-manager-improvement-actions.md) 、組織が管理するアクションです。 改善アクションを使用すると、コンプライアンス活動を一元化し、データ保護の規制と標準に合わせて調整できます。 各改善アクションでは、詳細な実装ガイダンスと、適切なソリューションへのリンクが提供されます。 改善アクションを組織内のユーザーに割り当て、実装とテストの作業を実行できます。 また、改善アクション内にドキュメント、メモ、およびレコードの状態の更新を保存できます。

### <a name="view-your-improvement-actions"></a>改善アクションの表示

コンプライアンス マネージャー ダッシュボードには、主な **改善アクションが表示されます。** すべての改善アクションを表示するには、ダッシュボードの [改善アクション] タブを選択して、改善アクション ページに移動します。 ダッシュボードの主要な改善アクションの一覧の下にある [すべての改善アクションを表示する] を選択して、改善アクション ページに移動することもできます。

[改善アクション] ページには、組織が管理しているすべての改善アクションが表示されます。 Microsoft によって管理されるアクションは、各評価内で表示できます (Microsoft アクションの詳細 [については、以下を参照してください](compliance-manager-assessments.md#microsoft-actions-tab))。

[改善アクション] ページにアクションの長いリストがある場合は、ビューをフィルター処理すると役立つ場合があります。 [アクション **] リスト** の右上隅にある [フィルター] を選択します。 [フィルター **] フライ** アウト ウィンドウが表示されたら、規制と標準、ソリューション、およびグループに基づいて条件を選択します。 右上隅の [グループ] **を選択して** 、ビューをカスタマイズできます。 ドロップダウン メニューから、グループ、ソリューション、カテゴリ、アクションの種類、または状態で表示する場合を選択します。

このページの既定のビューには、テストの状態が [合格] の改善アクションは **表示されません**。 テストに合格したアクションを表示するには、[フィルター] **フライアウト ウィンドウの [渡** された] ボックスをオンにします。 テストステータスが [合格] の **アクションのみ** スコアにカウントされます。 一部のアクションでは、 **保留中の更新ラベルが表示される場合があります。** 改善アクションの [更新について詳しくは、以下を参照してください](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)。

[改善アクション] ページには、各改善アクションの次のデータ ポイントが表示されます。

- **達成ポイント**: アクションを完了して使用可能な合計から達成されたポイント数
- **規制**: アクションに関連する規制または基準
- **グループ**: アクションを割り当てたグループ
- **ソリューション**: アクションの実行に移動できるソリューション
- **評価**: アクションを含む評価
- **カテゴリ**: 関連するデータ保護カテゴリ (情報の保護、デバイスの管理など)
- **テストの状態**:
    - **なし** – 状態の更新は記録されません
    - **評価されない** - テストが開始されていない
    - **合格** - 実装が正常にテストされました
    - **失敗した低リスク** - テストに失敗し、リスクが低い
    - **中規模リスクの失敗** - テストに失敗し、中程度のリスク
    - **失敗した高リスク** - テストに失敗し、リスクが高い
    - **スコープ外** – アクションは評価の対象ではなく、スコアに影響を与える
    - **手動テストの場合** は、アクションが実装されているが、テストされていない状態を示します。自動テストの場合は、アクションが自動化結果を待機中かどうかを示します。
    - **検出できない -** 自動状態を判断できない
    - **部分的にテスト済み** – 部分ポイントを与える自動スコアリング

**詳細については、「改善アクション**[に関する作業を割り当て、実行する方法」を参照してください](compliance-manager-improvement-actions.md)。

## <a name="solutions-page"></a>[ソリューション] ページ

[ソリューション] ページには、ソリューション別に整理された獲得ポイントと潜在的なポイントのシェアが表示されます。 このビューから残りのポイントと改善アクションを表示すると、より迅速に注意を払う必要があるソリューションを理解できます。

コンプライアンス マネージャー ダッシュボードの [ソリューション] タブ **を選択** して、ソリューション ページを探します。 ダッシュボードの右上 **のセクション** で、スコアに影響を与えるソリューションの下のすべてのソリューションを表示する を選択することもできます。

### <a name="filtering-your-solutions-view"></a>ソリューション ビューのフィルター処理

ソリューションのビューをフィルターするには、次の方法を使用します。

1. 評価 **リストの** 左上隅にある [フィルター] を選択します。
2. [フィルター **] フライ** アウト ウィンドウで、目的の条件 (標準と規制、ソリューション、アクションの種類、コンプライアンス マネージャー グループ、カテゴリ) の横にチェックを行います。
3. [適用] **ボタンを選択** します。 フィルター ウィンドウが閉じ、フィルター処理されたビューが表示されます。

また、評価リストの上にある [グループ] ドロップダウン メニューからグループ化の種類を選択して、グループ、製品、または規制別に評価を表示するビューを変更することもできます。

### <a name="taking-action-from-the-solution-page"></a>ソリューション ページからアクションを実行する

[ソリューション] ページには、改善アクションに接続されている組織のソリューションが表示されます。 次の表に、各ソリューションの全体的なスコアへの貢献度、そのソリューション内で達成および可能なポイント、およびスコアを上げ得るソリューションにグループ化された改善アクションの残りの数を示します。

この画面からアクションを実行するには、次の 2 つの方法があります。

1. 目的のソリューションの行の [残りのアクション] 列で、ハイパーリンクされた番号を選択します。 そのソリューションのテストされていない改善アクションを示す[改善アクション] 画面のフィルター処理されたビューが表示されます。

2. 目的のソリューションの行の [ソリューションを開く] **列で、[開** く] を **選択します**。 Microsoft 365 および Office 365 セキュリティ/コンプライアンス センターにソリューションまたは場所が表示され、推奨されるアクションを実行できます。

## <a name="assessments-page"></a>[評価] ページ

[評価] ページには、組織に [対](compliance-manager-assessments.md) して設定したすべての評価が一覧表示されます。 コンプライアンス スコア分母は、すべての追跡評価によって決まります。 評価を追加すると、改善アクション ページに改善アクションが表示され、コンプライアンス スコア分母が増加します。

ページ **の上部近** くのアクティブ化されたテンプレート カウンターには、組織で使用できるテンプレートの総数から現在使用されているアクティブな評価テンプレートの数が表示されます。 詳細については [、「テンプレートの種類](compliance-manager-templates.md#template-types-included-and-premium-active-and-inactive) 」を参照してください。

[評価] ページには、各評価に関する重要な情報が要約されています。

- **評価**: 評価の名前
- **状態**:
    - **Complete** - すべてのコントロールの状態が "渡された" か、少なくとも 1 つが渡され、残りは "スコープ外" になります。
    - **不完全** – 少なくとも 1 つのコントロールの状態が "failed"
    - **なし** - すべてのコントロールがテストされていない
    - **進行中 -** 改善アクションには、"進行中"、"部分与信"、または "検出されない" などの他の状態があります。
- **評価の** 進行状況 : 正常にテストされたコントロールの数によって測定される、完了に向けて行われた作業の割合
- **改善アクション**: コントロールの実装を満たす完了したアクションの数
- **Microsoft アクション**: Microsoft コントロールの実装を満たす完了したアクションの数
- **グループ**: 評価が属するグループの名前
- **製品**: 関連する Microsoft 365 サービス
- **規制**: 評価に適用される規制基準、ポリシー、または法律

### <a name="filtering-your-assessments-view"></a>評価ビューのフィルター処理

評価の表示をフィルターするには、次の処理を行います。

1. 評価 **リストの** 左上隅にある [フィルター] を選択します。
2. [フィルター **] フライ** アウト ウィンドウで、目的の条件を確認します。
3. [適用] **ボタンを選択** します。 フィルター ウィンドウが閉じ、フィルター処理されたビューが表示されます。

また、評価リストの上にある [グループ] ドロップダウン メニューからグループ化の種類を選択して、グループ、製品、または規制別に評価を表示するビューを変更することもできます。

### <a name="default-assessment"></a>既定の評価

既定では、[評価] ページにデータ [保護ベースライン](compliance-manager-assessments.md#data-protection-baseline-default-assessment) 評価が表示されます。 コンプライアンス マネージャーには、ビルド評価用の事前構築 [された](compliance-manager-templates-list.md) テンプレートもいくつか用意されています。

## <a name="assessment-templates-page"></a>[評価テンプレート] ページ

テンプレートは、コンプライアンス マネージャーで評価を作成するためのフレームワークです。 [評価テンプレート] ページには、テンプレートとキーの詳細の一覧が表示されます。 この一覧には、コンプライアンス マネージャーによって提供されるテンプレートと、組織が変更または作成したテンプレートが含まれます。 フィルターを適用して、認定、製品範囲、国、業界、作成者に基づいてテンプレートを検索できます。

ページ **の上部近** くのアクティブ化されたテンプレート カウンターには、組織で使用できるテンプレートの総数から現在使用されているアクティブな評価テンプレートの数が表示されます。 詳細については [、「テンプレートの種類](compliance-manager-templates.md#template-types-included-and-premium-active-and-inactive) 」を参照してください。

行からテンプレートを選択して詳細ページを表示します。このページには、テンプレートの説明と、認定、スコープ、およびコントロールの詳細に関する詳細情報が含まれます。 このページから、適切なボタンを選択して評価を作成したり、テンプレート データを Excel にエクスポートしたり、テンプレートを変更することができます。

**詳細については、「評価**[テンプレートを使用する方法」を参照してください](compliance-manager-templates.md)。
