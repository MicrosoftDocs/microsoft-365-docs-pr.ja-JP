# <a name="get-started-with-compliance-manager"></a>コンプライアンス マネージャーの使用を開始する

**この記事では、次の情報を参照してください。** この記事は、コンプライアンス マネージャーのセットアップに役立ちます。 コンプライアンス マネージャーにアクセスし **、役割** とアクセス許可を **設定** し、改善アクションの自動テストを **構成する方法について学習します**。 コンプライアンス マネージャー **ダッシュボードを** 確認し、メイン ページ (改善アクション ページ、ソリューション ページ、評価ページ、評価テンプレート ページ) を理解します。

## <a name="who-can-access-compliance-manager"></a>コンプライアンス マネージャーにアクセスできるユーザー

コンプライアンス マネージャーは、Office 365 ライセンスと Microsoft 365 ライセンスを持つ組織、および米国政府機関コミュニティ クラウド (GCC) Moderate および GCC High のお客様が利用できます。 評価の可用性と管理機能は、ライセンス契約によって異なっています。  [サービスの説明の詳細を表示します](https://go.microsoft.com/fwlink/?linkid=2132371)。

## <a name="before-you-begin"></a>開始する前に

組織の Microsoft 365 グローバル管理者が、コンプライアンス マネージャーにアクセスする最初のユーザーになる可能性があります。 コンプライアンス マネージャーに初めてアクセスする場合は、グローバル管理者がサインインし、以下に示すユーザーアクセス許可を設定することをお勧めします。

## <a name="sign-in"></a>サインイン

1. [Microsoft 365](https://compliance.microsoft.com/)コンプライアンス センターに移動し、Microsoft 365 グローバル管理者アカウントでサインインします。
2. 左側 **のナビゲーション ウィンドウで** コンプライアンス マネージャーを選択します。 コンプライアンス マネージャー ダッシュボードに [アクセスします](#understand-the-compliance-manager-dashboard)。

コンプライアンス マネージャーにアクセスする直接リンクは次のリンクです [https://compliance.microsoft.com/compliancemanager](https://compliance.microsoft.com/compliancemanager) 。

## <a name="set-user-permissions-and-assign-roles"></a>ユーザーのアクセス許可を設定し、ロールを割り当てる

コンプライアンス マネージャーは、役割ベースのアクセス制御 (RBAC) アクセス許可モデルを使用します。 ロールが割り当てられているユーザーだけがコンプライアンス マネージャーにアクセスできます。各ユーザーが許可するアクションは、ロールの種類によって [制限されます](#role-types)。

### <a name="where-to-set-permissions"></a>アクセス許可を設定する場所

組織のグローバル管理者の役割を持つユーザーは、COmpliance Manager のユーザーアクセス許可を設定できます。 アクセス許可は、Office 365 セキュリティ/コンプライアンス センター& Azure Active Directory (Azure AD) で設定できます。

> [!NOTE]
> 米国政府機関向けコミュニティ (GCC) High 環境のお客様は、Azure AD でコンプライアンス マネージャーのユーザーのアクセス許可とロールのみを設定できます。 Azure の手順とロールADについては、以下を参照してください。

Office 365 Security & コンプライアンス センターでアクセス許可を設定し、役割を割り当てるには、次の手順を実行します。

1. [Office 365 Security & コンプライアンス センター](https://protection.office.com/)に移動し、左側のナビゲーションで **[** アクセス許可] を選択します。

2. 1 人または複数のユーザーを追加する役割グループを見つけ、グループ名の左側にあるチェック ボックスをオンにします。 (以下の役割 [と関連する関数の一覧を参照してください](#role-types)。 役割グループ名は、役割名に似た名前です)。

3. そのグループのフライアウト ウィンドウで、Members ヘッダーの下の **[編集** ] を **選択** します。

4. [メンバー **の選択] を選択します**。 別のフライアウト ウィンドウが表示されます。

5. [ **+ 追加]** を選択して、グループに追加する 1 人または複数のユーザーを選択します。

6. 追加する名前の横にあるチェック ボックスをオンにし、下部にある [ **追加** ] ボタンを選択します。

7. ユーザーの割り当てが完了したら、[完了] を選択し、[保存]、次に **[閉** じる] の順に **選択します**。

## <a name="settings-for-automated-testing-and-user-history"></a>自動テストとユーザー履歴の設定

Microsoft 365 コンプライアンス センターのコンプライアンス マネージャーの設定を使用すると、改善のためのアクションの自動テストを有効または無効にできます。 また、この設定を使用すると、改善アクションに関連付けられたユーザーのデータを管理できます。また、改善アクションを別のユーザーに再割り当てする機能も含めます。  コンプライアンス マネージャーの設定にアクセスできるのは、グローバル管理者またはコンプライアンス マネージャー管理者の役割を持つユーザーのみです。

> [!NOTE]
> 自動テスト機能は、セキュリティ スコアがこれらの環境では使用できないので、GCC High 環境のお客様は利用できません。 GCC High のお客様は、改善アクションを手動で実装してテストする必要があります。

### <a name="set-up-automated-testing"></a>自動テストをセットアップする

コンプライアンス マネージャーの改善アクションの一部は [、Microsoft セキュア スコアによっても監視されます](https://docs.microsoft.com/microsoft-365/security/mtp/microsoft-secure-score)。 共同で監視されるアクションの自動テストを設定できます。つまり、セキュリティ スコアでアクションがテストおよび更新された場合、それらの結果はコンプライアンス マネージャーの同じアクションと同期され、コンプライアンス スコアにカウントされます。

コンプライアンス マネージャーを使い始めない組織では、自動テストが既定で有効になっています。 Microsoft 365 または Office 365 を初めて展開する場合、セキュア スコアがデータを完全に収集してコンプライアンス スコアに組み込むには、約 7 日間かかる場合があります。  自動テストを有効にすると、アクションのテスト日は更新されませんが、テストの状態は更新されます。 新しい評価が作成されると、スコアには Microsoft コントロール スコアとセキュア スコア統合が自動的に含まれます。

組織のグローバル管理者は、自動テストの設定をいつでも変更できます。 一般的な改善のための自動テストをオフにしたり、個々のアクションに対して有効にしたりすることができます。 自動テストの設定を変更するには、次の手順に従います。

#### <a name="to-manage-your-automated-testing-settings"></a>自動テスト設定を管理するには:

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. 設定ページで、コンプライアンス マネージャーを **選択します**。

3. 左側 **のナビゲーションから [自動** テスト] を選択します。

4. 該当するボタンを選択して、すべての改善アクションの自動テストを有効にするか、すべてのアクションでオフにするか、個別のアクションでオンにします。

5. [改善アクション **ごとに有効** にする] を選択すると、選択できるすべての改善アクションが一覧に表示されます。  自動的にテストするアクションの横にあるチェック ボックスをオンにします。

6. [保存 **] を** 選択して設定を保存します。 選択内容が保存されたという確認メッセージが画面の上部に表示されます。 エラー通知が表示された場合は、もう一度お試しください。

**注:** すべてのアクションの自動更新を有効またはオフにできるのは、グローバル管理者のみです。 コンプライアンス マネージャー管理者は、個々のアクションの自動更新を有効にできますが、すべてのアクションに対してグローバルに有効にできるのではありません。

### <a name="manage-user-history"></a>ユーザー履歴を管理する

[ **ユーザー履歴の管理]** 設定を使用すると、コンプライアンス マネージャーで改善アクションを実行したユーザーをすばやく特定できます。 改善アクションに関連付けられている特定可能なユーザー データには、実行された実装とテストの作業、アップロードしたドキュメント、および入力したメモが含まれます。 この種のデータの理解と取得は、組織独自のコンプライアンスニーズに必要な場合があります。

また、ユーザー履歴の設定では、すべての改善アクションをあるユーザーから別のユーザーに再割り当てすることもできます。

**ユーザー履歴の設定を見つけるには、次の方法を実行します。**

1. [Microsoft 365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. 設定ページで、コンプライアンス マネージャーを **選択します**。

3. 左側 **のナビゲーションから [ユーザー履歴の** 管理] を選択します。

[ **ユーザー履歴の管理** ] ページには、改善アクションに割り当てられているメール アドレス別のすべてのユーザーの一覧が表示されます。 [検索 **] ボタンを** 使用して、メール アドレスを入力して特定のユーザーをすばやく検索します。

各ユーザーのメール アドレスの右側にある [選択] ドロップダウン メニューには、レポートのエクスポート、改善アクションの再割り当て、履歴の削除を行うオプションがあります。 各オプションの詳細については、以下の各セクションを参照してください。

#### <a name="export-a-report-of-user-history-data"></a>ユーザー履歴データのレポートをエクスポートする

ユーザーに現在割り当てられている改善アクションの一覧を含む Excel ファイルをエクスポートできます。  レポートには、そのユーザーによってアップロードされた証拠ファイルも一覧表示されます。 この情報は、オープンな改善アクションを再割り当てするのに役立ちます。

レポートには、作成日の改善アクションの状態が反映されます。 これは、その状態または割り当てに対する以前のすべての変更の履歴レポートではありません (改善アクション ページからレポートをエクスポートする方法 [について説明します](compliance-manager-improvement-actions.md#export-a-report))。

**次の手順に従って、ユーザー別にレポートをエクスポートします。**

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. 設定ページで、コンプライアンス マネージャーを **選択します**。

3. 左側 **のナビゲーションから [ユーザー履歴** の管理] を選択します。

4. 目的のユーザーを検索するには、リストのメール アドレスを検索するか、[検索] を選択してユーザーのメール アドレスを入力します。

5. [選択 **] ドロップダウン メニュー** から [レポートのエクスポート] **を選択します**。

6. レポートの Excel ファイルが生成されると、そのファイルを開いてローカル コンピューターに保存できます。

#### <a name="reassign-improvement-actions-to-another-user"></a>改善アクションを別のユーザーに再割り当てする

あるユーザーから別のユーザーに改善アクションを再割り当てできます。 アクションを再割り当てしても、ドキュメントのアップロード履歴は変更されませんが、ドキュメントを最初にアップロードしたユーザーの名前は、改善アクション内に表示されなくなりました。

**次の手順に従って、改善アクションを別のユーザーに再割り当てします。**

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. 設定ページで、コンプライアンス マネージャーを **選択します**。

3. 左側 **のナビゲーションから [ユーザー履歴** の管理] を選択します。

4. リストのメール アドレスを検索するか、[検索] を選択してそのユーザーのメール アドレスを入力して、ユーザーを検索します。

5. [選択] **ドロップダウン メニューで** 、[改善アクションの再割り当 **て] を選択します**。 [ **改善アクションの再割り当て** ] フライアウト ウィンドウが表示されます。

6. [検索ユーザー **] フィールド** に、改善アクションを割り当てるユーザーの名前または電子メール アドレスを入力 *します*。

7. [改善アクション] の下に目的のユーザーの名前が表示された場合は、ユーザーを選択し、[アクションの割り当て]**を選択します**。

8. 再割り当てが完了すると、フライアウト ウィンドウに確認メッセージが表示され、前のユーザーからのすべての改善アクションが新しいユーザーに再割り当てされたという確認メッセージが表示されます。 再割り当てエラー通知が表示された場合は、ウィンドウを閉じてからもう一度やり直してください。 フライアウト ウィンドウを閉じるには、[完了] を選択 **します**。

新しい割り当て先は、改善アクションに割り当てられたメールを受信します。 メールには、改善アクションの詳細ページへの直接リンクが含まれている。
 
 > [!NOTE]
> 保留中の更新があるアクションを再割り当てすると、再割り当て後に更新が受け入れられると、再割り当てメール内のアクションへの直接リンクが壊れる可能性があります。 この問題を解決するには、更新プログラムが受け入れられると、ユーザーにアクションを再割り当てします。 改善アクションの [更新について詳しくは、以下を参照してください](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)。

#### <a name="delete-user-history"></a>ユーザー履歴を削除する

ユーザーの履歴を削除すると、ユーザーは改善アクションの所有者として削除され、コンプライアンス マネージャーの他のすべてのフィールドから名前が削除されます。 ユーザーの履歴を削除すると、ユーザーが所有する改善アクションには、新しいユーザーが割り当てられるまで **[** 割り当て済み] の値は表示されません。 改善アクションにアップロードされたドキュメントには、 **削除されたユーザー** の名前の代りで削除されたユーザーが表示されます。 ユーザー履歴の削除は永続的です。

ユーザーの履歴を削除するには、次の手順を実行します。

1. Microsoft  [365](https://compliance.microsoft.com/)コンプライアンス センターの任意の場所から左側のナビゲーションの [設定] を選択します。

2. 設定ページで、コンプライアンス マネージャーを **選択します**。

3. 左側 **のナビゲーションから [ユーザー履歴** の管理] を選択します。

4. リストのメール アドレスを検索するか、[検索] を選択してそのユーザーのメール アドレスを入力して、ユーザーを検索します。

5. From the **Select drop-down** menu, choose **Delete history**.

6. ユーザーの履歴の完全な削除を確認するウィンドウが表示されます。 削除を続行するには、[履歴の削除] **を選択します**。 履歴を削除せずに残す場合は、[キャンセル] を選択 **します**。

7. [ユーザー履歴の管理]ページに戻り、ユーザーの履歴が削除されたという確認メッセージが上部に表示されます。

## <a name="understand-the-compliance-manager-dashboard"></a>コンプライアンス マネージャー ダッシュボードについて

コンプライアンス マネージャー ダッシュボードは、現在のコンプライアンス体制を一目で確認できる設計です。

![コンプライアンス マネージャー - ダッシュボード](../media/compliance-manager-dashboard.png "コンプライアンス マネージャー ダッシュボード")

### <a name="overall-compliance-score"></a>全体的なコンプライアンス スコア

コンプライアンス スコアは、上部に目立つように表示されます。 主要なデータ保護の標準と規制に対応する改善のためのアクションを完了するために達成可能なポイントに基づくパーセンテージを示します。 Microsoft が [管理する Microsoft](compliance-manager-assessments.md#microsoft-actions-tab)アクションからのポイントは、コンプライアンス スコアにもカウントされます。

コンプライアンス マネージャーに初めてアクセスする場合、最初のスコアは [Microsoft 365 データ保護ベースラインに基づいて作成されます](compliance-manager-assessments.md#data-protection-baseline-default-assessment)。 このベースライン評価は、すべての組織で利用できます。これは、一般的な業界の規制と基準を含む一連のコントロールです。 コンプライアンス マネージャーは、既存の Microsoft 365 ソリューションをスキャンし、現在のプライバシーとセキュリティ設定に基づいて初期評価を行います。 組織に関連する評価を追加すると、スコアが意味を持つものになります。

**詳細については、「コンプライアンス**[スコアの計算方法」を参照してください](compliance-score-calculation.md)。

### <a name="key-improvement-actions"></a>主要な改善アクション

このセクションでは、コンプライアンス スコア全体に最も良い影響を与える上で、今すぐ実行できる最も良い改善アクションを示します。 [ **すべての改善アクションを表示] を選択** して、改善アクション ページに移動します。

### <a name="solutions-that-affect-your-score"></a>スコアに影響を与えるソリューション

このセクションでは、スコアに良い影響を与える可能性のある改善アクションと、それらのソリューションの未解決の改善アクションの数を含むソリューションについて説明します。 [すべての **ソリューションの表示] を選択** して、ソリューション ページにアクセスします。

### <a name="compliance-score-breakdown"></a>コンプライアンス スコアの詳細

このセクションでは、2 つの異なる方法でスコアの詳細を確認できます。

- **カテゴリ**: "情報の保護" や "デバイスの管理" など、データ保護カテゴリ内の全体的なスコアの割合を示します。
- **Assessment:** GDPR や NIST 800-53 などの特定のコンプライアンスとデータ保護の標準、規制、法律に関する評価を管理する進捗状況の割合を示します。

### <a name="filtering-your-dashboard-view"></a>ダッシュボード ビューのフィルター処理

ダッシュボード ビューをフィルター処理して、特定の規制と標準、ソリューション、アクションの種類、評価グループ、またはデータ保護カテゴリに関連する項目のみを表示できます。 この方法でビューをフィルター処理すると、ダッシュボードのスコアもフィルター処理され、フィルター条件に基づいて、可能な合計ポイントから得たポイント数が表示されます。

フィルターを適用するには:

1. ダッシュボード **の右上** にある [フィルター] を選択します。
2. [フィルター] フライアウト ウィンドウからフィルター **条件** を選択し、[適用] を選択 **します**。

フィルターを適用すると、スコアがリアルタイムで調整されます。 コンプライアンス スコアのパーセンテージとブレークダウン情報、および改善のためのアクションとソリューションは、フィルター条件の対象となるデータにのみ関係します。 コンプライアンス マネージャーからサインアウトすると、サインイン時にフィルター処理されたビューが残ります。

フィルターを削除するには:

- コンプライアンス スコア **の上にある** [適用されたフィルター] 見出しで、削除する個々のフィルターの横にある **[X]** を選択します。または
- ダッシュボード **の右上** にある [フィルター] を選択し、[フィルター]フライアウト ウィンドウで [フィルターのクリア]**を選択します**。

## <a name="improvement-actions-page"></a>[改善アクション] ページ

[改善アクションは](compliance-manager-improvement-actions.md) 、組織によって管理されるアクションです。 改善アクションを操作すると、コンプライアンス活動を一元化し、データ保護の規制と基準に準拠できます。 各改善アクションは、詳細な実装ガイダンスと、適切なソリューションへのリンクを提供します。 改善のためのアクションを組織内のユーザーに割り当て、実装とテストの作業を実行できます。 また、改善アクション内にドキュメント、メモ、および状態の更新を記録できます。

### <a name="view-your-improvement-actions"></a>改善アクションを表示する

コンプライアンス マネージャー ダッシュボードには、主要な **改善アクションが表示されます。** すべての改善アクションを表示するには、ダッシュボードの [改善アクション] タブを選択すると、改善アクション ページが表示されます。 ダッシュボードの主要な改善アクションの一覧の下にある [すべての改善アクションを表示] を選択して、改善アクションページに移動することもできます。

[改善アクション] ページには、組織で管理されている改善アクションすべてが表示されます。 Microsoft によって管理されるアクションは、各評価内で表示できます (Microsoft のアクションの詳細 [をご確認ください](compliance-manager-assessments.md#microsoft-actions-tab))。

改善アクション ページにアクションの長いリストがある場合は、ビューをフィルター処理すると便利です。 アクション **リスト** の右上隅にある [フィルター] を選択します。 [ **フィルター]** フライアウト ウィンドウが表示されたら、規制と標準、ソリューション、およびグループに基づいて条件を選択します。 右上隅にある [グループ] **を選択して** 、ビューをカスタマイズできます。 ドロップダウン メニューから、グループ、ソリューション、カテゴリ、アクションの種類、または状態で表示を選択します。

このページの既定のビューには、テストの状態が [合格] の改善のためのアクションは表示 **されません**。 テストに合格したアクションを表示するには、[フィルター] フライアウト ウィンドウの **[Passed]** ボックスをオンにします。 テスト状態が [合格] の **アクションだけが** スコアにカウントされます。 一部の操作では、保留中 **の更新ラベルが表示される場合があります。** 改善アクションの [更新について詳しくは、以下を参照してください](compliance-manager-improvement-actions.md#accepting-updates-to-improvement-actions)。

[改善アクション] ページには、各改善アクションの次のデータ ポイントが表示されます。

- **達成ポイント**: アクションを完了することによって、利用可能な合計から得たポイント数
- **規制**: アクションに関連する規制または基準
- **グループ**: アクションを割り当てたグループ
- **ソリューション**: アクションを実行するために移動できるソリューション
- **Assessment :** アクションを含む評価
- **カテゴリ**: 関連するデータ保護カテゴリ (情報の保護、デバイスの管理など)
- **テストの状態**:
    - **なし** - 状態の更新は記録されません
    - **評価されない** - テストが開始されていない
    - **合格** - 実装が正常にテストされました
    - **失敗した低リスク** - テストに失敗し、リスクが低い
    - **中程度のリスクの失敗** - テストに失敗し、中程度のリスク
    - **失敗した高リスク** - テストに失敗し、リスクが高い
    - **対象範囲外** – アクションは評価の対象ではなく、スコアには影響を与えない
    - **検出する -** 手動テストの場合は、アクションが実装されているが、テストが行えないを示します。自動テストの場合、アクションが自動化の結果を待機中を示します
    - **検出できない -** 自動状態を判断できない
    - **部分的にテスト済み** – 部分的なポイントを与える自動スコアリング

**詳細については、「改善アクションを**[割り当て、作業を実行する方法」を参照してください](compliance-manager-improvement-actions.md)。

## <a name="solutions-page"></a>[ソリューション] ページ

[ソリューション] ページには、獲得したポイントと潜在的なポイントの共有がソリューション別に表示されます。 このビューから残りのポイントと改善アクションを表示すると、どのソリューションにすぐに注意を払う必要があるのか理解するのに役立ちます。

コンプライアンス マネージャー ダッシュボードの [ソリューション] タブを選択して、ソリューション ページを見つける。 ダッシュボードの右上 **の** セクションで、スコアに影響を与えるソリューションの下のすべてのソリューションを表示を選択することもできます。

### <a name="filtering-your-solutions-view"></a>ソリューション ビューのフィルター処理

ソリューションのビューをフィルター処理するには:

1. 評価 **リスト** の左上隅にある [フィルター] を選択します。
2. [ **フィルター]** フライアウト ウィンドウで、目的の条件 (標準と規制、ソリューション、アクションの種類、コンプライアンス マネージャー グループ、カテゴリ) の横にチェックを配置します。
3. [適用] **ボタンを選択** します。 フィルター ウィンドウが閉じ、フィルター処理されたビューが表示されます。

また、評価リストの上にある [グループ] ドロップダウン メニューからグループ化の種類を選択して、グループ、製品、規制別に評価を表示するビューを変更することもできます。

### <a name="taking-action-from-the-solution-page"></a>ソリューション ページからアクションを実行する

[ソリューション] ページには、改善のためのアクションに関連付けられている組織のソリューションが表示されます。 表には、各ソリューションの全体的なスコアへの貢献度、そのソリューション内で達成および可能なポイント、およびスコアを上げられる可能性があるソリューションにグループ化された改善アクションの残りの数が示されています。

この画面からアクションを実行するには、次の 2 つの方法があります。

1. 目的のソリューションの行の [残りのアクション] 列で、ハイパーリンク番号を選択します。 そのソリューションの未テストの改善アクションが表示された、フィルター処理された改善アクション画面が表示されます。

2. 目的のソリューションの行の [ソリューションを開く] 列 **で** 、[開く] を選択 **します**。 Microsoft 365 および Office 365 のセキュリティ/コンプライアンス センターに、推奨されるアクションを実行できるソリューションまたは場所が表示されます。

## <a name="assessments-page"></a>[Assessment] ページ

[評価] ページには、組織 [に設定した](compliance-manager-assessments.md) 評価の一覧が表示されます。 コンプライアンス スコア分母は、すべての追跡評価によって決まります。 評価を追加すると、改善のためのアクションのページに改善のためのアクションが表示され、コンプライアンス スコア分母が増加します。

評価ページには、各評価に関する重要な情報が要約されます。

- **Assessment**: 評価の名前
- **Status**:
    - **Complete** - すべてのコントロールの状態が "passed" である、または少なくとも 1 つが渡され、残りは "スコープ外"
    - **不完全** - 少なくとも 1 つのコントロールの状態が "失敗" である
    - **なし** - すべてのコントロールがテストされていない
    - **進行中** - 改善アクションには、"進行中"、"部分与信"、"未検出" などのその他の状態があります
- **評価の** 進行状況 : 正常にテストされたコントロールの数で測定された完了までの作業の割合
- **改善アクション**: コントロールの実装を満たす完了したアクションの数
- **Microsoft のアクション**: Microsoft コントロールの実装を満たす完了したアクションの数
- **グループ**: 評価が属するグループの名前
- **製品**: 関連付けられた Microsoft 365 サービス
- **規制**: 評価に適用される規制基準、ポリシー、または法律

### <a name="filtering-your-assessments-view"></a>評価ビューのフィルター処理

評価のビューをフィルター処理するには::

1. 評価 **リスト** の左上隅にある [フィルター] を選択します。
2. [ **フィルター]** フライアウト ウィンドウで、目的の条件を確認します。
3. [適用] **ボタンを選択** します。 フィルター ウィンドウが閉じ、フィルター処理されたビューが表示されます。

また、評価リストの上にある [グループ] ドロップダウン メニューからグループ化の種類を選択して、グループ、製品、規制別に評価を表示するビューを変更することもできます。

### <a name="default-assessment"></a>既定の評価

既定では、データ保護ベースラインの評価[](compliance-manager-assessments.md#data-protection-baseline-default-assessment)が [評価] ページに表示されます。 コンプライアンス マネージャーには、評価を作成する [事前に作成](compliance-manager-templates-list.md) されたテンプレートもいくつか用意されています。

## <a name="assessment-templates-page"></a>評価テンプレート ページ

テンプレートは、コンプライアンス マネージャーで評価を作成するためのフレームワークです。 [評価テンプレート] ページには、テンプレートと主要な詳細の一覧が表示されます。 この一覧には、コンプライアンス マネージャーによって提供されるテンプレートと、組織が変更または作成したテンプレートが含まれます。 フィルターを適用して、認定、製品範囲、国、業界、およびそれを作成したユーザーに基づいてテンプレートを検索できます。

行からテンプレートを選択して詳細ページを表示します。このページには、テンプレートの説明と、認定、範囲、およびコントロールの詳細に関する詳細情報が含まれます。 このページから適切なボタンを選択して、評価の作成、テンプレート データの Excel へのエクスポート、またはテンプレートの変更を行います。

**詳細については、「評価**[テンプレートを使用する方法」を参照してください](compliance-manager-templates.md)。
