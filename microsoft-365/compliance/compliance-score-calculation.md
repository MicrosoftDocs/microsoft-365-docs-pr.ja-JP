 - <span data-ttu-id="bb330-170">**必須アクションは** 、意図的または誤ってバイパスできません。</span><span class="sxs-lookup"><span data-stu-id="bb330-170">**Mandatory actions** can't be bypassed, either intentionally or accidentally.</span></span> <span data-ttu-id="bb330-171">必須アクションの例は、パスワードの長さ、複雑さ、有効期限の要件を設定する、一般に管理されたパスワード ポリシーです。</span><span class="sxs-lookup"><span data-stu-id="bb330-171">An example of a mandatory action is a centrally managed password policy that sets requirements for password length, complexity, and expiration.</span></span> <span data-ttu-id="bb330-172">ユーザーがシステムにアクセスするには、次の要件を満たす必要があります。</span><span class="sxs-lookup"><span data-stu-id="bb330-172">Users must follow these requirements to access the system.</span></span>
  
 - <span data-ttu-id="bb330-173">**任意のアクションは** 、ポリシーを理解し、遵守するためにユーザーに依存します。</span><span class="sxs-lookup"><span data-stu-id="bb330-173">**Discretionary actions** rely upon users to understand and adhere to a policy.</span></span> <span data-ttu-id="bb330-174">たとえば、ユーザーがコンピューターを離れるときにユーザーにロックを要求するポリシーは、ユーザーに依存する任意のアクションです。</span><span class="sxs-lookup"><span data-stu-id="bb330-174">For example, a policy requiring users to lock their computer when they leave it is a discretionary action because it relies on the user.</span></span>
  
### <a name="preventative-detective-and-corrective-actions"></a><span data-ttu-id="bb330-175">予防、探偵、および修正のアクション</span><span class="sxs-lookup"><span data-stu-id="bb330-175">Preventative, detective, and corrective actions</span></span>
  
 - <span data-ttu-id="bb330-176">**予防措置は、特定** のリスクに対処します。</span><span class="sxs-lookup"><span data-stu-id="bb330-176">**Preventative actions** address specific risks.</span></span> <span data-ttu-id="bb330-177">たとえば、暗号化を使用して保存中の情報を保護すると、攻撃や侵害に対する予防措置です。</span><span class="sxs-lookup"><span data-stu-id="bb330-177">For example, protecting information at rest using encryption is a preventative action against attacks and breaches.</span></span> <span data-ttu-id="bb330-178">職務の分離は、利益相反を管理し、詐欺から保護するための予防措置です。</span><span class="sxs-lookup"><span data-stu-id="bb330-178">Separation of duties is a preventative action to manage conflict of interest and guard against fraud.</span></span>
  
 - <span data-ttu-id="bb330-179">**検出アクションは、** システムを積極的に監視して、リスクを表す不規則な条件や動作、または侵入や侵害を検出するために使用できる動作を識別します。</span><span class="sxs-lookup"><span data-stu-id="bb330-179">**Detective actions** actively monitor systems to identify irregular conditions or behaviors that represent risk, or that can be used to detect intrusions or breaches.</span></span> <span data-ttu-id="bb330-180">たとえば、システム アクセスの監査と特権管理アクションが含まれます。</span><span class="sxs-lookup"><span data-stu-id="bb330-180">Examples include system access auditing and privileged administrative actions.</span></span> <span data-ttu-id="bb330-181">規制コンプライアンス監査は、プロセスの問題を見つけるために使用される検出アクションの一種です。</span><span class="sxs-lookup"><span data-stu-id="bb330-181">Regulatory compliance audits are a type of detective action used to find process issues.</span></span>
  
- <span data-ttu-id="bb330-182">**修正アクションは** 、セキュリティ インシデントの悪影響を最小限に抑え、直ちに影響を軽減する是正措置を取り、可能な場合は損害を取り消します。</span><span class="sxs-lookup"><span data-stu-id="bb330-182">**Corrective actions** try to keep the adverse effects of a security incident to a minimum, take corrective action to reduce the immediate effect, and reverse the damage if possible.</span></span> <span data-ttu-id="bb330-183">プライバシー インシデント対応は、侵害後にシステムの損傷を制限し、システムを運用状態に復元する是正措置です。</span><span class="sxs-lookup"><span data-stu-id="bb330-183">Privacy incident response is a corrective action to limit damage and restore systems to an operational state after a breach.</span></span>
