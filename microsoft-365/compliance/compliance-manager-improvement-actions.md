# <a name="assign-and-complete-improvement-actions-in-compliance-manager"></a>コンプライアンス マネージャーで改善アクションを割り当て、完了する

**この記事では、次の情報を参照してください。** この記事では、改善アクションを使用 **してコンプライアンス ワークフローを管理する** 方法について説明します。 実装とテスト、 **更新プログラムの管理、レポート** のエクスポートを行う改善のためのアクションを **割** り当てる方法について説明 **します**。

## <a name="manage-compliance-workflows-with-improvement-actions"></a>改善アクションを使用してコンプライアンス ワークフローを管理する

改善アクションは、コンプライアンス アクティビティを一元化します。 各改善アクションは、データ保護の規制と基準に準拠するのに役立つ詳細な実装ガイダンスを提供します。 組織内のユーザーにアクションを割り当て、実装とテストの作業を実行できます。 アクション内にドキュメント、メモ、および状態の更新を記録できます。

すべての改善アクションが[改善アクション] ページに一覧表示されます。 改善アクションの [表示について詳しくは、以下をご覧ください](compliance-manager-setup.md#improvement-actions-page)。

## <a name="improvement-actions-details-page"></a>改善アクションの詳細ページ

各改善アクションには、現在の状態、関連する標準と規制要件、推奨される実装ガイダンスを示す詳細ページがあります。 [技術的なアクション](compliance-score-calculation.md#technical-and-non-technical-actions) には、 **実装** に適したソリューションに進む [今すぐ起動] リンクが含まれます。 実装とテストのドキュメントを、改善のためのアクションの詳細ページに直接添付できます。

改善アクションの詳細ページを表示するには:

1. 改善アクションページに移動します。
2. 目的の改善アクションの行を選択すると、詳細ページが開きます。

次または以前の改善アクションを一覧で簡単に表示するには、画面の右上隅にある上矢印または下矢印を選択します。 改善アクション ページでリストをフィルター処理した場合、上または下に移動すると、フィルター処理されたリスト内の次の項目に移動します。

## <a name="assign-improvement-actions"></a>改善アクションを割り当てる

改善アクションの実装作業を開始するには、自分で作業を実行するか、別のユーザーに割り当てる必要があります。 割り当てられたユーザーは次の場合があります。

- ビジネス ポリシーの所有者
- IT 担当者
- タスクを実行する責任を持つ別の従業員

適切な割り当て先を特定したら、作業を実行するのに十分な [コンプライアンス](compliance-manager-setup.md#set-user-permissions-and-assign-roles) マネージャーの役割を持っている必要があります。 次の手順に従って、改善アクションを割り当てします。

1. 改善アクションの詳細ページで、画面 **の左上** のセクションの近くの [状態の編集] を選択します。

2. [状態の編集] フライアウト ウィンドウで、[割り当て先] ボックスを選択して、ユーザーの候補ユーザーの一 **覧** を表示します。 一覧からユーザーを選択するか、ユーザーを割り当てるユーザーのメール アドレスを入力します。

3. [保存 **して閉じる] を選択します**。 割り当てられたユーザーには、改善アクションが割り当てられたと説明する電子メールと、改善アクションへの直接リンクが送信されます。 (注: 米国政府機関コミュニティ (GCC) の高いユーザーは、アクションが割り当てられたときにメールを受信しません。

割り当てられたユーザーは、推奨されるアクションを実行できます。

#### <a name="assign-multiple-improvement-actions-to-a-single-user"></a>1 人のユーザーに複数の改善アクションを割り当てる

次の手順に従って、1 人のユーザーに複数の改善アクションを割り当てできます。

1. [改善アクション] ページに移動します。
2. 改善アクションの名前の左側にある領域を選択します。 そのアクションを選択したことを示すラウンド チェック アイコンが表示されます。 割り当てるすべてのアクションを確認します。
3. 改善アクション **の表の** 上部にある [ユーザーに割り当て] リンクを選択します。
4. ポップアップ ウィンドウが表示されます。 [割り **当て先** ] フィールドで、アクションを割り当てるユーザーの名前の入力を開始します。 また、候補のユーザーの一覧から選択できます。
5. [割り当て先 **] フィールドに** 割り当て先の名前を設定した後、[割り当て] を選択 **します**。
6. その後、[改善のためのアクション] ページが表示され、割り当てたアクションの新しい割り当て先が一覧表示されます。

## <a name="perform-work-and-store-documentation"></a>作業とストアのドキュメントを実行する

実装とテストに関連するファイルとメモを[メモとドキュメント] **セクションに直接アップロード** できます。 この環境は、セキュリティで保護された集中管理されたリポジトリであり、コンプライアンスの標準と規制を満たすコントロールの満足度を実証するのに役立ちます。 読み取り専用アクセス権を持つすべてのユーザーは、このセクションのコンテンツを読み取りできます。 編集権限を持つユーザーだけがファイルをアップロードおよびダウンロードし、メモを入力または編集できます。

「 **メモとドキュメント」セクションには** 、アップロードされたドキュメント、実装に関するメモ、テスト ノート、および追加のメモのフィールドが含まれています。

#### <a name="uploaded-documents"></a>アップロードされたドキュメント

- 関連する **ファイルをアップロードするには、[ドキュメントの** 管理] を選択します。
- [ドキュメントの管理] フライアウト ウィンドウが開いたら、[ドキュメントの追加] を選択し、システムからファイルを選択します。 受け入れられるファイルの種類:
    - ドキュメント (.doc、.xls、.ppt、.txt、.pdf)
    - 画像 (.jpg、.png)
    - ビデオ (.mkv)
    - 圧縮ファイル (.zip、.rar)
- ウィンドウでファイルが解決したら、[閉じる] **を選択して**、添付ファイルを自動的に保存します。 アップロードされたドキュメントの下にファイルが **一覧表示されます**。
- ドキュメントをダウンロードまたは削除するには、ドキュメントのリストの下から [ドキュメントの管理] を選択します。 フライアウト ウィンドウで、ドキュメント行を選択して強調表示し、[ダウンロード] または [削除 **] を** 選択 **します**。

#### <a name="implementation-notes-test-notes-and-additional-notes"></a>実装に関するメモ、テスト ノート、その他のメモ

- これらの 3 つのフィールドにメモを追加するには、これらのフィールドの下にある [実装メモの編集] を選択します。
- フライアウト ウィンドウが開いたら、テキスト フィールドにメモを入力し、[保存して閉じる] **を選択します**。
- ノートを編集するには、[実装メモの **編集] を選択し**、編集を行い、[保存して **閉じる] を選択します**。

ノート フィールドに文字制限はありません。 改善アクションの詳細ページから簡単に表示および編集できるよう、メモを簡潔にすることをお勧めします。

## <a name="change-improvement-action-status"></a>改善アクションの状態を変更する

各改善アクションについて、実装の状態と日付、テストの状態と日付を記録できます。 実装 **および** テスト **の状態** フィールドは、割り当てられたユーザーではなく、編集権限を持つ任意のユーザーが編集できます。

改善アクションの状態を編集するには、詳細ページの左上のセクションで [状態の編集] を選択します。 使用可能なフィールドと状態オプションを次に示します。

- **実装の状態**
    - **実装されていません** - アクションはまだ実装されていません
    - **実装** - 実装されたアクション
    - **代替実装** - 他のサード パーティ製ツールを使用した場合、または Microsoft の推奨事項に含まれていない他のアクションを実行した場合は、このオプションを選択します
    - **計画済** み - アクションの実装が計画されている
    - **スコープ外** – アクションは組織に関連するのではないし、スコアに貢献しない
- **実装日**: 実装状態が "実装" または "代替実装" の場合に選択できます。
- **テストの状態**: 実装状態が "実装" または "代替実装" の場合に選択できます。
    - **評価されない** – アクションがテストされていない
    - **合格** - 評価者によって実装が検証されました
    - **失敗した低リスク** - テストに失敗し、リスクが低い
    - **中程度のリスクの失敗** - テストに失敗し、中程度のリスク
    - **失敗した高リスク** - テストに失敗し、リスクが高い
    - **スコープ外** – アクションは評価の対象から外れ、スコアに影響を及びない
- **テスト日**: カレンダーのポップアップを切り替え、日付を選択する

一般的なアクションはグループ間で同期されます。 同じグループ内の 2 つの異なる評価が、自分が管理する改善アクションを共有する場合、アクションの実装の詳細または状態に対して行った更新は、グループ内の他の評価の同じアクションに自動的に同期されます。 この同期を使用すると、1 つの改善アクションを実装し、複数の規制でいくつかの要件を満たします。

## <a name="assign-improvement-action-to-assessor-for-completion"></a>完了のために評価者に改善アクションを割り当てる

作業を完了し、テストを実施し、証拠をアップロードしたら、次の手順は、検証のために評価者に改善アクションを割り当てる方法です。 評価者は作業を検証し、ドキュメントを調べ、適切なテスト状態を選択します。

**テストの状態が "Passed"** に設定されている場合: アクションは完了し、達成されたポイントは達成された最大ポイントを示します。 その後、ポイントはコンプライアンス スコア全体にカウントされます。

**テストの状態が "失敗"** に設定されている場合: アクションは要件を満たしません。評価者は、追加の作業のために適切なユーザーに割り当て戻します。

## <a name="accepting-updates-to-improvement-actions"></a>改善アクションの更新を受け入れる

更新プログラムが改善アクションに使用できる場合、その名前の横に通知が表示されます。 更新プログラムを受け入れるか、後で延期することができます。

#### <a name="what-causes-an-update"></a>更新の原因

更新は、スコア、自動化、または範囲に関連する変更があるときに発生します。 変更には、規制の変更に基づく改善のための新しいガイダンスが含まれる場合や、製品の変更が原因である可能性があります。 組織が管理する改善アクションだけが更新通知を受け取る。

#### <a name="where-youll-see-assessment-update-notifications"></a>評価更新通知が表示される場所

改善アクションが更新された場合、その名前の横に[改善アクション] ページの [保留中の更新プログラム] ラベルが表示され、関連する評価の詳細ページに表示されます。

改善アクションの詳細ページに移動し、上部のバナーの[更新の確認] ボタンを選択して、変更に関する詳細を確認し、更新プログラムを承諾または延期します。

#### <a name="review-update-to-accept-or-defer"></a>更新プログラムを確認して承諾または延期する

改善アクションの **詳細ページ** から [更新の確認] を選択すると、画面の右側にフライアウト ウィンドウが表示されます。 フライアウト ウィンドウには、影響を受け取った評価やスコアと範囲の変更など、更新に関する重要な詳細が表示されます。

[ **更新を承諾する]** を選択して、改善アクションへのすべての変更を受け入れる。 **承諾された変更は永続的です**。

> [!NOTE]
> アクションの更新を受け入れる場合は、このアクションの他のバージョンまたはインスタンスに対する更新も受け入れる必要があります。 更新プログラムは、技術的なアクションのためにテナント全体に伝達され、非技術的なアクションについてはグループ全体に伝達されます。

[キャンセル] **を** 選択した場合、更新プログラムは改善アクションに適用されません。 ただし、更新プログラムに同意するまでは、保留中 **の** 更新通知が引き続き表示されます。

**更新プログラムの受け入れをお勧めする理由**

更新プログラムを受け入れて、ソリューションの使用に関する最新のガイダンスを確実に得て、適切な改善アクションを実行して、認定の要件を満たします。

**更新プログラムを延期する理由**

改善アクションを含む評価を完了している場合は、更新プログラムに同意する前に、その評価に関する作業が完了済みである必要があります。 レビュー更新のフライアウト ウィンドウで [キャンセル]を選択すると、更新プログラムを後で延期できます。

#### <a name="accept-all-updates-at-once"></a>すべての更新プログラムを一度に受け入れる

複数の更新プログラムを使用し、一度にすべて受け入れる場合は、改善アクションの表の上部にある [すべての更新プログラムを受け入れる] リンクを選択します。 更新するアクションの数を示すフライアウト ウィンドウが表示されます。 [更新プログラムの **承諾] ボタンを** 選択して、すべての更新プログラムを適用します。

改善のためのアクション ページに戻った場合、ページの上部にメッセージが表示され、更新が完了するためにページを更新するメッセージが表示されることがあります。

## <a name="export-a-report"></a>レポートをエクスポートする

画面 **の** 左上隅にある [エクスポート] を選択して、すべての改善アクションと、改善アクション ページに表示されるフィルター カテゴリを含む Excel ワークシートをダウンロードします。
