# <a name="the-email-entity-page"></a>[電子メール] エンティティ ページ

**この記事の内容:**
- [電子メール エンティティ ページにアクセスする](#reach-the-email-entity-page)
- [電子メール エンティティ ページの読み取り](#read-the-email-entity-page)

Office 365 (または MDO) E5 用の Microsoft Defender と MDO P1 と P2 の管理者は、[電子メール] エンティティ ページを使用してメールを 360 度表示 **できます**。 この移動先メール ページは、脅威エクスプローラーの 「メールの詳細」のフライアウトで配信される情報 [を強化するために作成されました](threat-explorer-views.md)。

## <a name="reach-the-email-entity-page"></a>電子メール エンティティ ページにアクセスする

既存の Office セキュリティ/コンプライアンス センター (protection.office.com) または新しい Microsoft 365 セキュリティ センター (security.microsoft.com) のいずれかを使用すると、電子メール エンティティ ページを表示して使用できます。

|中央  |URL  |ナビゲーション  |
|---------|---------|---------|
|セキュリティとコンプライアンス |protection.office.com | 脅威管理> エクスプローラー   |
|Microsoft 365 セキュリティ センター |security.microsoft.com | メール& コラボレーション > エクスプローラー |

脅威エクスプローラーで、調査しているメールの件名を選択します。 そのメールのメール のフライアウトの上部にゴールドのバーが表示されます。 新しいページへのこの招待は、「新しい電子メール エンティティ ページをリッチ データで試してみる....」と読みます。 新しいページを表示する場合に選択します。

:::image type="content" source="../../media/email-entities-1-navigation-to-ee.png" alt-text="You will see a gold banner with the words *Try out our new email entity page with enriched data* to navigate to the new experience.":::

:::image type="content" source="../../media/email-entities-2-eep.png" alt-text="電子メール エンティティ ページのこの図は、表示される見出しに焦点を当てしています。電子メール ヘッダーがここに表示されます。":::

> [!NOTE]
> このページの表示と使用に必要なアクセス許可は、脅威エクスプローラーを表示する場合と同じです。 管理者は、グローバル管理者またはグローバル閲覧者、またはセキュリティ管理者またはセキュリティ閲覧者のメンバーである必要があります。

## <a name="read-the-email-entity-page"></a>電子メール エンティティ ページの読み取り

構造は、読みやすく、一目で確認できる構造です。 ページの上部にあるさまざまなタブを使用すると、より詳細に調査できます。 レイアウトのしくみを次に示します。

1. 最も必要なフィールドは、フライアウトの左側にあります。これらの詳細は "固定" です。つまり、フライアウトの残りの部分で移動するタブに関係なく、左側にアンカーされます。

    :::image type="content" source="../../media/email-entities-3-left-panel.png" alt-text="左側が強調表示されている電子メール エンティティ ページの図。メール配信に関するタイトルと事実は、ここです。":::

2. 右上隅には、メールに対して実行できるアクションがあります。 エクスプローラーで実行できるアクションは、電子メール エンティティ ページでも利用できます。

    :::image type="content" source="../../media/email-entities-5-preview.png" alt-text="今回は、*right* 側が強調表示されている電子メール エンティティ ページの図。[メールのプレビュー] や [検疫に移動] のようなアクションがここに表示されます。":::

3. ページの残りの部分を並べ替え、より詳細な分析を実行できます。 メール検出の詳細、メール認証の状態、ヘッダーを確認します。 この領域はケース バイ ケースで表示されますが、これらのタブの情報は任意のメールで利用できます。

    :::image type="content" source="../../media/email-entities-4-middle-panel.png" alt-text="このページのメイン パネルには、電子メール ヘッダーと認証の状態が表示されます。":::

### <a name="use-email-entity-page-tabs"></a>電子メール エンティティ ページタブを使用する

エンティティ ページの上部にあるタブを使用すると、電子メールを効率的に調査できます。

1. **タイムライン**: (脅威エクスプローラーのタイムラインごとに) メールのタイムライン ビューには、メールで発生した配信後イベントへの元の配信が表示されます。 配信後の操作がないメールの場合、ビューには元の配信行がタイムライン ビューに表示されます。 次のようなイベント: ゼロアワー自動消去 (ZAP)、修復、URL クリック、et cetera(システム、管理者、ユーザーなど) は、発生した順序でここに表示されます。
2. **分析**: 分析は、管理者がメールを詳細に分析するのに役立つフィールドを示します。 管理者が検出、送信者/受信者、メール認証の詳細について理解する必要がある場合は、[分析] タブを使用する必要があります。添付ファイルと URL のリンクは、このページの [関連するエンティティ] の下にも表示されます。 ここでは、添付ファイルと特定された脅威の両方に番号が付きます。クリックすると、[添付ファイル] ページと [URL] ページに直接アクセスできます。 このタブには、電子メール ヘッダーを表示する View *ヘッダー オプションも含まれています*。 管理者は、わかりやすくするために、メール ヘッダーの詳細とメイン パネルの情報を並べて比較できます。
3. **添付** ファイル : 電子メールで見つかった添付ファイルを調べ、添付ファイルに関するその他の詳細を確認します。 現在、表示される添付ファイルの数は 10 に制限されています。 悪意のある添付ファイルに関するデトレーションの詳細もここに示されています。
4. **URL :** このタブには、電子メール内の URL と URL に関するその他の詳細が一覧表示されます。 URL の数は現在 10 に制限されています。ただし、これらの 10 は悪意のある URL を最初に表示するために *優先順位が付けられています*。 [Prioritization] を使用すると、時間と推測の作業が節約されます。 悪意のある URL として検出され、その URL がここにも表示されます。
5. **類似メール**: このタブには、ネットワーク メッセージ *ID と* このメールに固有の受信者の組み合わせに似たすべてのメールが一覧表示されます。 類似性は、メッセージ *の本文にのみ* 基づいております。 メールを 「類似」として分類する決定には、添付ファイルの考慮事項は含 *めされません*。

## <a name="new-to-the-email-entity-page"></a>電子メール エンティティ ページの新機能

この電子メール エンティティ ページには、新しい機能が追加されています。 一覧を次に示します。

### <a name="email-preview-for-cloud-mailboxes"></a>クラウド メールボックスのメール プレビュー
メールがクラウドにまだ存在する場合、管理者はクラウド メールボックスでメールをプレビューできます。 (管理者またはユーザーによる) 回復可能な削除、または ZAP (検疫) の場合、メールはクラウドの場所に存在しなくなりました。 その場合、管理者はそれらの特定のメールをプレビューできない可能性があります。 ドロップされた電子メール、または配信が失敗した電子メールは、実際にはメールボックスに配信されません。 その結果、管理者もこれらのメールをプレビューできません。

> [!WARNING]
>メールのプレビューには、管理者に割り当てるには、 ***Preview** _ という特別な役割が必要です。 この役割を追加するには *、_Permissions*& roles * > Email **& collaboration roles** in *security.microsoft.com*, or **Permissions** in *protection.office.com*. プレビューの ***役割*** を任意の役割グループに追加するか、組織内の管理者が脅威エクスプローラーで作業できる役割グループのコピーを追加します。

### <a name="detonation-details"></a>デトレーションの詳細

これらの詳細は、メールの添付ファイルと URL に固有です。

ユーザーには、メールボックス内で見つかった既知の悪意のある添付ファイルやハイパーリンクに関する、強化された分析の詳細 (分析チェーン、分析の概要、スクリーンショット、観察された動作の詳細など) が表示され、添付ファイルまたは URL が悪意があると見なされ、分析された理由をユーザーが理解するのに役立ちます。
 
- *分析チェーン*: 1 つのファイルまたは URL の分析によって、複数の分析をトリガーできます。 分析チェーンは、分析のパスを追跡します。このパスには、元の悪意のあるファイルや、その分析によって引き起こされたその他すべてのファイルまたは URL が含されます。 これらの URL または添付ファイルが電子メールに直接存在しない場合がありますが、その分析を含めるのは、ファイルまたは URL が悪意のあるファイルと見なされた理由を判断する上で重要です。
- *デトレーションの概要*: 次の情報が提供されます。
    - デトレーション時間の範囲。
    - 添付ファイル (URL) の確認。
    - 分析中に調べる他のエンティティである関連情報 (ファイル番号、URL、IPs、またはドメイン)。
- *デトレーションのスクリーンショット*: これは、デトレーション プロセス中に取得されたスクリーンショットを示しています。
- *分析の詳細*: これらは、分析中に行った各プロセスの正確な動作の詳細です。

:::image type="content" source="../../media/email-entities-6-detonation-page.png" alt-text="チェーン、概要、分析の詳細、および見出し *Deep Analysis* の下のスクリーンショットを示す分析の概要のスクリーンショット。":::

### <a name="other-innovations"></a>その他の技術革新

*タグ*: ユーザーに適用されるタグです。 ユーザーが受信者の場合、管理者には受信者タグが *表示* されます。 同様に、ユーザーが送信者の場合は *送信者タグ* です。 これは、電子メール エンティティ ページの左側 (固定と表示され、ページにアンカーが付いたパーツ内) に表示されます。

*最新の配信場所*: 最新の配信場所は、ZAP のようなシステムアクションの後に電子メールが届いた場所、または削除済みアイテムに移動する、完了のような管理者アクションが送信された場所です。 最新の配信場所は、メッセージの現在の場所を管理者に通知することを *目的と* していない。 たとえば、ユーザーがメッセージを削除したり、メッセージをアーカイブに移動したりした場合、配信場所は更新されません。 ただし、システムアクションが実行され、場所が更新された場合 (ZAP のように、メールが検疫に移動する場合)、これにより最新の配信場所が検疫に更新されます。

*電子メールの* 詳細 : [分析] タブで使用できる電子メールをより深く理解するために *必要な* 詳細。

- *Exchange トランスポート ルール (ETRs またはメールフロー* ルール) : これらのルールはトランスポート層のメッセージに適用され、フィッシングおよびスパムの検出より優先されます。 これらは Exchange 管理センターでのみ作成および変更できますが、ETR がメッセージに適用される場合は、ETR 名と GUID がここに表示されます。 追跡目的で有用な情報。
    
- *システムオーバーライド*: これは、(脅威と検出の技術に関する) システムによって提供される配信場所を上書きして、メッセージの配信場所に対して例外を作成する手段です。
    
- *迷惑メール メールボックス ルール*: '迷惑メール' は、すべてのメールボックスで既定で有効になっている非表示の受信トレイ ルールです。
    - メールボックスで迷惑メール ルールが有効になっている場合、Exchange Online Protection (EOP) は、いくつかの条件に従ってメッセージを迷惑メールに移動できます。 移動は、スパム フィルターの決まったアクション *[* 迷惑メール] フォルダーにメッセージを移動するか、メールボックスの [受信拒否リスト] に基づいて行います。 迷惑メール ルールを無効にすると、メールボックスの差出人セーフ リストに基づいて迷惑メール フォルダーにメッセージを配信できません。
    - メールボックスで迷惑メール ルールが無効になっている場合、EOP はスパム フィルターの確認アクションに基づいてメッセージを [迷惑メール] フォルダーに移動できません。メッセージを [迷惑メール] フォルダーまたはメールボックスのセーフ リスト コレクションに移動します。
    
- *一括準拠レベル (BCL)*: メッセージのバルク苦情レベル (BCL)。 BCL が高いほど、バルク メール メッセージが苦情を生成する可能性が高くなります (メールがスパムである可能性が高い場合の自然な結果)。
    
- *Spam Confidence Level (SCL)*: メッセージの Spam Confidence Level (SCL)。 値が高いほど、メッセージがスパムである可能性が高くなります。

- *ドメイン名*: 送信者のドメイン名です。
    
- *Domain Owner*: 送信ドメインの所有者を指定します。
    
- *ドメインの* 場所 : 送信ドメインの場所を指定します。
    
- *Domain Created Date*: 送信ドメインの作成日を指定します。 新しく作成されたドメインは、他のシグナルが疑わしい動作を示している場合に注意が必要になる可能性があります。

*電子メール認証*: Microsoft 365 で使用される電子メール認証方法には、SPF、DKIM、および DMARC が含まれます。

- Sender Policy Framework **(SPF):** メッセージの SPF チェックの結果について説明します。 次の値を使用できます。
    - パス (IP アドレス): メッセージの SPF チェックが渡され、送信者の IP アドレスが含まれます。 送信者のドメインに代わってメールを送信または中継する許可がクライアントに与えられています。
    - 失敗 (IP アドレス): メッセージの SPF チェックに失敗し、送信者の IP アドレスが含まれています。 hard fail と呼ばれることもあります。
    - Softfail (理由): SPF レコードは、送信は許可されていないが、移行中としてホストを指定しました。
    - ニュートラル: SPF レコードは、IP アドレスが送信を承認されているかどうかをアサートしないかどうかを明示的に示します。
    - なし: ドメインに SPF レコードが存在しないか、SPF レコードが結果として評価されます。
    - Temperror: 一時的なエラーが発生しました。 たとえば、DNS エラーです。 同じチェックが後で成功する場合があります。
    - Permerror: 永続的なエラーが発生しました。 たとえば、ドメインに不正な形式の SPF レコードがある場合などです。

- DomainKeys Identified Mail (**DKIM**):
    - Pass: 渡されたメッセージの DKIM チェックを示します。
    - 失敗 (理由): メッセージの DKIM チェックが失敗した理由を示します。 たとえば、メッセージが署名されていない場合、署名を認証できない場合などです。
    - なし: メッセージが署名されていないかどうかを示します。 これは、ドメインに DKIM レコードがあるかどうかや、DKIM レコードが結果を評価しない (このメッセージが署名されていない点のみ) ことを示しますが、これらを示さない場合もあります。

- ドメイン ベースのメッセージ認証、レポート、準拠 **(DMARC):**
    - Pass: 渡されたメッセージの DMARC チェックを示します。
    - Fail: メッセージの DMARC チェックが失敗したかどうかを示します。
    - Bestguesspass: ドメインの DMARC TXT レコードが存在しないが、存在していた場合は、メッセージの DMARC チェックがパスしていたことを示します。
    - なし: DNS の送信ドメインに対して DMARC TXT レコードが存在しないかどうかを示します。

*複合認証*: これは、Microsoft 365 が SPF、DKIM、DMARC のような電子メール認証を組み合わせて、メッセージが本物かどうかを判断するために使用される値です。 評価の基礎 *としてメールの From:* ドメインを使用します。
